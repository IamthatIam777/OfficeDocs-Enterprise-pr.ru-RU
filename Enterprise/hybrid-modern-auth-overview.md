## <a name="what-is-modern-authentication"></a>Что такое современная проверка подлинности?
<a name="BKMK_WhatisModAuth"> </a>

При общении между клиентом (например, ноутбуком или телефоном) и сервером Корпорация Майкрософт использует фразу "современная проверка подлинности".
  
Современная проверка подлинности — это типичный термин для методов проверки подлинности и авторизации, а также некоторые меры безопасности, зависящие от политик доступа, с которыми вы уже можете ознакомиться. В него входят:
  
- **Способы проверки**подлинности: многофакторная проверка подлинности; Проверка подлинности на основе сертификата клиента.
    
- **Методы проверки подлинности**: реализация Open Authorization (OAuth) корпорации Майкрософт. 
    
- **Политики условного доступа**: Управление мобильными приложениями (MAM) и условный доступ к Azure Active Directory. 
    
Управление удостоверениями пользователей с помощью современной проверки подлинности дает администраторам множество различных средств по обеспечению безопасности ресурсов и предоставляет более безопасные методы управления удостоверениями для локальных (Exchange и Skype для бизнеса), гибридной среды Exchange и Skype для бизнеса смешанных и разделенных доменных сценариев.
  
Имейте в виду, что так как Skype для бизнеса работает в тесном взаимодействии с Exchange, то в соответствии с современным состоянием проверки подлинности Exchange пользователи клиента Skype для бизнеса будут видеть результаты проверки подлинности при входе в систему. Это также применимо, если используется гибридная среда Skype для бизнеса с разделенным доменом. Кроме того, тип гибридной среды Skype для бизнеса, поддерживающий использование современной проверки подлинности, часто называется "Split-Domain" (в разделенном домене, у вас есть как Skype для бизнеса Online, так и Skype для бизнеса в локальной системе, и пользователи размещены в обоих расположениях).
  
> [!IMPORTANT]
> Знаете ли вы, что в августе 2017 для всех новых клиентов Office 365, использующих Skype для бизнеса Online и Exchange Online, включена ли современная проверка подлинности по умолчанию? Существующие клиенты не будут изменять состояние MA по умолчанию, но все новые клиенты автоматически поддерживают расширенный набор функций идентификации, перечисленных выше. Чтобы проверить состояние MA для Skype для бизнеса Online, вы можете использовать Skype для бизнеса Online PowerShell с учетными данными глобального администратора. Запустите `Get-CsOAuthConfiguration` , чтобы проверить выходные данные параметра – клиентадалаусоверриде. Если для-Клиентадалаусоверриде задано "разрешено", ваша современная проверка подлинности включена. 
  
## <a name="what-changes-when-i-use-modern-authentication"></a>Какие изменения используются при использовании современной проверки подлинности?
<a name="BKMK_WhatChanges"> </a>

При использовании современной проверки подлинности в локальной среде Skype для бизнеса или Exchange Server вы по-прежнему *проверяете подлинность* пользователей в локальной среде, а также историю *авторизации* доступа к ресурсам (например, файлам или сообщениям электронной почты). Вот почему современная проверка подлинности относится к обмену данными между клиентом и сервером, действия, выполняемые во время настройки Евостс (служба маркеров безопасности, используемая Azure AD), устанавливаются как сервер проверки подлинности для Skype для бизнеса и Exchange Server локально. 
  
Изменение Евостс позволяет вашим локальным серверам использовать OAuth (выдачу маркера) для авторизации клиентов, а также для локального использования методов безопасности, общих для облака (например, многофакторной проверки подлинности). Кроме того, Евостс выдает маркеры, которые позволяют пользователям запрашивать доступ к ресурсам, не предоставляя свой пароль в качестве части запроса. Независимо от того, где находятся пользователи (из Интернета или локальной среды), и независимо от того, в каком расположении размещен требуемый ресурс, Евостс становится ядром авторизации пользователей и клиентов после настройки современной проверки подлинности.
  
Вот пример того, что я имею в виду. Если клиент Skype для бизнеса должен получить доступ к Exchange Server для получения сведений о календаре от имени пользователя, для этого используется библиотека проверки подлинности Active Directory (ADAL). ADAL это библиотека кода, предназначенная для обеспечения доступности защищенных ресурсов в вашем каталоге для клиентских приложений с помощью маркеров безопасности OAuth. ADAL работает с OAuth для проверки утверждений и маркеров Exchange (а не паролей), чтобы предоставить пользователю доступ к ресурсу. В прошлое полномочия в транзакции, такие как этот сервер, который знает, как проверять утверждения пользователей и выдать необходимые маркеры, могут быть локальными службами маркеров безопасности или даже службами федерации Active Directory. Однако современная проверка подлинности централизует эту службу с помощью Azure Active Directory (Azure AD) в облаке.
  
Это также означает, что несмотря на то, что среда Exchange Server и Skype для бизнеса могут быть полностью настроены в локальной среде, сервер авторизации будет доступен в сети, а локальная среда должна иметь возможность создания и обслуживания подключения к Office. 365 подписка в облаке (и экземпляр Azure Active Directory, которую ваша подписка использует в качестве каталога).
  
Что не меняется? Независимо от того, используете ли вы гибридный домен или локальную среду Skype для бизнеса и Exchange Server, все пользователи должны сначала пройти проверку подлинности *локально* . В гибридной реализации современной проверки подлинности Линкдисковери и автообнаружения указывает на локальный сервер. 
  
> [!IMPORTANT]
> Если вам нужно знать конкретные топологии Skype для бизнеса, поддерживаемые в MA, это [описано здесь](https://technet.microsoft.com/library/mt803262.aspx).
  
## <a name="check-the-modern-authentication-status-of-your-on-premises-environment"></a>Проверка состояния современной проверки подлинности в локальной среде
<a name="BKMK_CheckStatus"> </a>

Так как современная проверка подлинности изменяет сервер авторизации, используемый при использовании служб OAuth/S2S, необходимо знать, включена ли современная проверка подлинности для среды Skype для бизнеса и Exchange. Вы можете проверить состояние серверов Exchange или Skype для бизнеса в локальной среде, выполнив `Get-CSOAuthConfiguration` команду в PowerShell. Если команда возвращает пустое свойство ' Оауссерверс ', то современная проверка подлинности отключена.

## <a name="what-else-do-i-need-to-know-before-i-begin"></a>Что еще нужно знать перед началом работы?
<a name="BKMK_Whatelse"> </a>

1. Все сценарии для локальных серверов включают настройку современной проверки подлинности в локальной среде (на самом деле для Skype для бизнеса существует список поддерживаемых топологий), чтобы сервер, отвечающий за проверку подлинности и авторизацию, был включен в Microsoft Cloud ( Служба маркеров безопасности AAD, называемая "Евостс"), и обновление Azure Active Directory (AAD) об URL-адресах или пространствах имен, используемых локальной установкой Skype для бизнеса или Exchange. Таким образом, локальные серверы принимают зависимость от облачных служб Майкрософт. Выполнение этого действия может рассматриваться как Настройка "гибридная проверка подлинности".
