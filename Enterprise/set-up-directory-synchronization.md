# <a name="set-up-directory-synchronization-for-office-365"></a>Настройка синхронизации каталогов для Office 365
Office 365 использует службы управления удостоверения пользователя на основе облака Azure Active Directory для управления пользователями. Также можно интегрировать локальной Active Directory с Azure AD, синхронизацию локальной среды с Office 365. После настройки синхронизации можно указать, кто имеет свои происходят в Azure AD или в локальном каталоге проверки подлинности пользователей.
  
## <a name="office-365-directory-synchronization"></a>Синхронизация службы каталогов Office 365
Можно использовать либо синхронизированного identity или федеративных удостоверений между локальной организацией и Office 365. С синхронизированной удостоверения для управления пользователями на локальную и Azure AD, при использовании тот же пароль в облаке как локальный проходят проверку подлинности. Это наиболее распространенные сценарии синхронизации службы каталогов. Сквозной проверки подлинности или федеративных удостоверений позволяет управлять пользователями на локальную и локального каталога проходят проверку подлинности. Федеративное удостоверение требуется дополнительная настройка и позволяет пользователям только входа в один раз. Для получения дополнительных сведений прочитайте [Общие сведения об Office 365 удостоверения и Azure Active Directory](about-office-365-identity.md).
  
## <a name="want-to-upgrade-from-windows-azure-active-directory-sync-dirsync-to-azure-active-directory-connect"></a>Хотите выполнить обновление до синхронизации Windows Azure Active Directory (DirSync) для Azure Active Directory подключения?
Если в настоящее время с помощью DirSync и хотите обновить, не забывайте также через посещать [azure.com](https://azure.com) для [инструкции по обновлению](https://go.microsoft.com/fwlink/p/?LinkId=733240).

## <a name="to-set-up-directory-synchronization"></a>Настройка синхронизации службы каталогов
1. Войдите Центр администрирования Office 365 и выберите **пользователей,** \> **Активных пользователей** на левой панели навигации. 
2. В центре администрирования Office 365 на странице **активных пользователей** выберите ** более ** \> **синхронизации службы каталогов**.
    
    ![Выберите в меню Дополнительные синхронизации службы каталогов](media/dc6669e5-c01b-471e-9cdf-04f5d44e1c4b.png)
  
3. На ** является синхронизации каталога непосредственно для вас? ** страницы, два варианта первого **1-10**и **11 – 50** результатов в «в зависимости от размера организации рекомендуется создание и управление пользователями в облаке. С помощью синхронизации службы каталогов сделает более сложных настроек. Перейдите к активных пользователей, чтобы добавить пользователей.» 
    
    - Тем не менее, можно тем не менее, продолжить настройку синхронизации службы каталогов, выбрав **Продолжить здесь** в нижней части страницы. 
    
    - При выборе варианта два последних варианта, **51 250** или **251 или более поздней версии**, настройки синхронизации будет рекомендуем синхронизации службы каталогов. Нажмите кнопку **Далее** для продолжения. 
    
    ![Нажмите Далее, чтобы продолжить настройку синхронизации службы каталогов](media/359a1eb9-99ae-4b5b-a413-4de53037cceb.png)
  
4. На **синхронизации на локальный каталог с облаком**, ознакомьтесь с информацией и Дополнительные сведения, выберите сведения ссылка на дополнительные почтовый: [Подготовка для подготовки пользователей с помощью синхронизации службы каталогов в Office 365](prepare-for-directory-synchronization.md)и нажмите кнопку Далее ** **. 
    
5. На странице " **проверим каталоге** " Обзор требований для автоматическую проверку наличия каталога. Если вы не отвечает требованиям, нажмите кнопку **Далее** \> **Начать сканирование**. Если не удается выполнить все требования по-прежнему можно продолжить, выбрав **Продолжить вручную**.
    
    ![Нажмите кнопку Далее или продолжить вручную на проверим страница каталога](media/af4a6bd5-13aa-4bfa-9751-4464a32ca8db.png)
  
6. Если выбран параметр для сканирования свои каталоги, выберите **Начать сканирование** на странице **Оценка настройки синхронизации каталогов** . 
    
    Следуйте инструкциям, чтобы загрузить и запустить сканирование.
    
7. По завершении сканирования возврата в мастер установки и нажмите кнопку **Далее** для просмотра результатов сканирования. 
    
8. Проверка домены, как указано на странице **Проверка владения домены** . Подробные сведения содержатся в разделе [Создание записей DNS для Office 365 при самостоятельном управлении записями DNS-записей](https://support.office.com/article/b0f3fdca-8a80-4e8e-9ef3-61e8a2a9ab23).
    
    > [!IMPORTANT]
    > После добавления записи TXT для проверки, что вы владеете вашего домена не перейдите к следующему шагу добавления пользователей в мастере доменов. Синхронизации каталогов добавит пользователей для вас. 
  
    Вернуться на страницу **Microsoft Office 365** и выберите команду **Обновить**
    
    ![После проверки домены, выберите команду Обновить](media/9b5fb593-5ff7-49f0-80d0-18e36d39d669.png)
  
9. На странице **домены будете готовы** нажмите кнопку **Далее**.
    
10. На странице **Очистка среды** при необходимости следуйте инструкциям по загрузке IDFix для проверки Active Directory. Нажмите кнопку **Далее** для продолжения. 
    
11. На ** запустите Azure Active Directory подключение ** выберите **загрузить** , чтобы установить мастер подключения Azure AD. 
    
    > [!NOTE]
    > На этом этапе будет находиться в мастере подключения Azure AD. Убедитесь, что оставьте страница мастера синхронизации службы каталогов, вы находились при открытии в браузере, поэтому вы можете вернуться к его после завершения действия подключить Azure AD. 
  
    После Azure AD подключить мастер установлен автоматически открывать. Можно также открыть с настольного компьютера, сайта установки по умолчанию. Следуйте инструкциям мастера зависит от используемого сценария:
    
  - Для синхронизации службы каталогов с помощью хэш-функции синхронизации паролей используйте [Azure AD подключиться с express параметры](https://go.microsoft.com/fwlink/p/?LinkID=698537).
    
  - Для нескольких лесов, сквозной проверки подлинности, федеративных удостоверений и параметры единого входа можно используйте [настраиваемые установки Azure AD подключение](https://go.microsoft.com/fwlink/p/?LinkId=698430).
    
    Выберите пункт **Настройка** на странице **Параметров Express** для использования этих параметров. 
    
12. После завершения мастера подключения Azure AD возврата в мастер **Установки Office 365** и следуйте инструкциям на **Убедитесь в том, синхронизации, работает как ожидается страницы**. Нажмите кнопку **Далее** для продолжения. 
    
13. Ознакомьтесь с инструкциями на ** активировать пользователей ** страницы и нажмите кнопку **Далее**.
    
14. На странице **вы все программы установки** нажмите кнопку **Готово** . 
    
## <a name="assign-licences-to-synchronized-users"></a>Назначение лицензий пользователям синхронизированного

Сведения содержатся в разделе [Назначение лицензий для пользователей в Office 365 для бизнеса](https://support.office.com/article/997596b5-4173-4627-b915-36abac6786dc).
