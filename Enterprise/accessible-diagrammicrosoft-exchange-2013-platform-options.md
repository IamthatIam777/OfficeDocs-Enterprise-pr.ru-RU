# <a name="accessible-diagram---microsoft-exchange-2013-platform-options"></a>Открытая диаграмма — варианты платформы Microsoft Exchange 2013

**Сводка.** Эта статья — удобная текстовая версия схемы "Варианты платформы для Microsoft Exchange 2013", которая доступна в разделе [Технические схемы](https://go.microsoft.com/fwlink/?LinkID=519139&amp;clcid=0x409).
  
На этом плакате представлена информация о развертывании Exchange Online и Exchange Server, которая необходима лицам, принимающим деловые решения (BDM), и архитекторам. Плакат содержит: 
  
- Сравнение четырех доступных вариантов платформы для Exchange 2013: облачной службы Exchange Online (Office 365), гибридного развертывания Exchange, локального сервера Exchange Server и службы Exchange с размещением у поставщика. 
    
- Описания трех новых или обновленных компонентов Exchange 2013. 
    
## <a name="comparison-of-four-different-deployments-for-the-exchange-2013-platform"></a>Сравнение четырех различных вариантов развертывания платформы Exchange 2013

В сравнении по каждому варианту развертывания представлена следующая информация: 
  
- обзор различных компонентов развертывания; 
    
- преимущества каждого из вариантов развертывания; 
    
- требования лицензирования; 
    
- необходимые архитектурные задачи; 
    
- обязанности ИТ-специалистов по внедрению каждого варианта развертывания. 
    
### <a name="overview"></a>Обзор

#### <a name="exchange-online-office-365"></a>Exchange Online (Office 365)

Больше эффективности и меньше затрат благодаря Office 365.
  
На сопутствующей диаграмме представлена служба Exchange Online с клиентом Azure Active Directory, которая синхронизирует имена и пароли учетных записей между локальной средой доменных служб Active Directory (AD DS) и клиентом Azure Active Directory. Для единого входа необходимы службы федерации Active Directory (AD FS). 
  
Описание компонентов и функций:
  
- Обслуживанием серверов и серверного программного обеспечения занимается Майкрософт.
    
- Обширный набор компонентов Exchange Server 2013 как облачной службы.
    
- Регулярное обновление компонентов.
    
- Служба Exchange Online Protection (EOP) для защиты от спама и вредоносных программ.
    
- Гарантия высокой доступности 99,9 % времени по условиям соглашения об уровне обслуживания (SLA).
    
- Синхронизация каталогов, в том числе паролей, между локальными доменными службами Active Directory (AD DS) и клиентом Azure Active Directory. Для единого входа необходимы службы федерации Active Directory (AD FS).
    
#### <a name="exchange-hybrid"></a>Гибридное развертывание Exchange

Пользуйтесь преимуществами Office 365, сохранив локальный сервер Exchange Server.
  
На сопутствующей диаграмме представлен вариант Office 365 с Exchange Online, где некоторые пользователи размещены локально, а некоторые  в сети. Кроме того, на диаграмме представлен клиент Azure Active Directory, который синхронизирует имена и пароли учетных записей между локальной средой доменных служб Active Directory (AD DS) и клиентом Azure Active Directory.
  
Описание компонентов и функций:
  
- Одни пользователи размещены локально, другие — в сети. Пользователи совместно используют общее адресное пространство электронной почты.
    
- Используется существующая инфраструктура Exchange Server.
    
- Постепенный переход с локальной среды Exchange на Exchange Online согласно вашему расписанию.
    
- Интеграция с другими приложениями Office 365, в том числе Lync Online и SharePoint Online.
    
#### <a name="exchange-server-on-premises"></a>Локальный сервер Exchange Server

Вы можете спроектировать и обслуживать собственную инфраструктуру Exchange Server 2013.
  
На сопутствующей диаграмме представлена инфраструктура Exchange Server с локальной средой доменных служб Active Directory (AD DS), где пользователи размещены локально.
  
Описание компонентов и функций:
  
- Максимальная степень контроля и настройки для вашей конфигурации.
    
- Независимость от сходства сеансов на уровне балансировки нагрузки.
    
- Высокая доступность и устойчивость сайта за счет групп обеспечения доступности баз данных (DAGs).
    
- Средства управления доступностью для высокого уровня взаимодействия с пользователями.
    
- Использование существующего оборудования и инфраструктуры системы хранения данных.
    
#### <a name="provider-hosted-exchange"></a>Exchange с размещением у поставщика

Вы можете передать обработку рабочих нагрузок Exchange Server поставщику решений Exchange Server.
  
На сопутствующей диаграмме представлена среда Exchange Server, которую обслуживает внешний поставщик.
  
Описание компонентов и функций:
  
- Обслуживанием серверов и серверного программного обеспечения занимается ваш поставщик.
    
- Планирование, определение размеров, масштабирование и техническое обслуживание инфраструктуры Exchange Server поручаются вашему поставщику.
    
- Эксплуатационным обслуживанием занимается ваш поставщик.
    
- Набор компонентов Exchange зависит от версии программного обеспечения, развернутого вашим поставщиком.
    
### <a name="benefits-of-implementing-each-deployment-option"></a>Преимущества каждого из вариантов развертывания

#### <a name="exchange-online-office-365"></a>Exchange Online (Office 365)

Этот вариант наиболее подходит:
  
- для организаций, которым нужно снизить эксплуатационные расходы на локально развернутые службы Exchange;
    
- организаций, которые намерены воспользоваться другими предложениями Office 365, такими как SharePoint Online и Lync Online.
    
#### <a name="exchange-hybrid"></a>Гибридное развертывание Exchange

Этот вариант наиболее подходит:
  
- для ускорения перехода с локальной среды Exchange на Exchange Online;
    
- поддержки удаленных сайтов без вложений в создание инфраструктуры филиалов;
    
- транснациональных корпораций с дочерними компаниями, которым необходимо локальное размещение данных.
    
#### <a name="exchange-server-on-premises"></a>Локальный сервер Exchange Server

Этот вариант наиболее подходит:
  
- для решений с широкими возможностями настройки;
    
- устаревших решений со сторонними компонентами, которые работают на аппаратном и программном обеспечении, не поддерживаемом службой Exchange Online;
    
- организаций, регулярно проходящих процедуры контроля управления данными, по причине чего необходимо локальное размещение данных;
    
- организаций, которым необходимо сохранить полный контроль над платформой и решением.
    
#### <a name="provider-hosted-exchange"></a>Exchange с размещением у поставщика

Этот вариант наиболее подходит:
  
- для организаций, которым нужны функции Exchange Server без необходимости самостоятельно развертывать и обслуживать платформу;
    
- организаций, которым необходимы специальные варианты поддержки;
    
- специальных решений и интеграции с настраиваемыми приложениями от поставщика;
    
- устаревших решений со сторонними компонентами, которые работают на аппаратном и программном обеспечении, не поддерживаемом службой Exchange Online.

### <a name="architecture-tasks"></a>Архитектурные задачи

В этом разделе перечислены архитектурные задачи для каждого варианта развертывания.
  
#### <a name="exchange-online-office-365"></a>Exchange Online (Office 365)

- Создание плана и проекта синхронизации каталогов.
    
- Обеспечение достаточной пропускной способности сети и надежного подключения через брандмауэры, прокси-серверы, шлюзы и каналы глобальной сети (WAN).
    
#### <a name="exchange-hybrid"></a>Гибридное развертывание Exchange

Помимо выполнения архитектурных задач для Office 365 и локальной среды, необходимо следующее:
  
- Принять решение о предоставлении возможности единого входа.
    
- Принять решение о направлении входящих сообщений электронной почты через локальную среду либо через службу Exchange Online Protection.
    
#### <a name="exchange-server-on-premises"></a>Локальный сервер Exchange Server

- Разработка топологии Exchange.
    
- Расчет емкости для серверного оборудования.
    
- Разработка топологии маршрутизации сообщений.
    
- Разработка схемы балансировки нагрузки для серверов клиентского доступа.
    
- Создание плана поддержания высокой доступности за счет групп обеспечения доступности баз данных.
    
#### <a name="provider-hosted-exchange"></a>Exchange с размещением у поставщика

Необходимо убедиться, что ваш поставщик располагает достаточной пропускной способностью сети и надежным подключением через брандмауэры, прокси-серверы, шлюзы и каналы глобальной сети (WAN).
  
### <a name="it-pro-responsibilities"></a>Обязанности ИТ-специалистов

В этом разделе перечислены обязанности ИТ-специалистов по внедрению каждого варианта развертывания.
  
#### <a name="exchange-online-office-365"></a>Exchange Online (Office 365)

- Внедрение плана синхронизации каталогов.
    
- Планирование внутренних и внешних записей DNS и маршрутизации.
    
- Настройка прокси-сервера или брандмауэра для IP-адреса Office 365 и требований URL-адреса.
    
- Администрирование учетных записей пользователей и настроек Exchange Online.
    
#### <a name="exchange-hybrid"></a>Гибридное развертывание Exchange

Помимо выполнения ИТ-специалистами Office 365 и локальной среды своих обязанностей, необходимо следующее:
  
- Настройка служб федерации Active Directory (AD FS) для единого входа (при необходимости).
    
- Настройка сертификатов Exchange для безопасных соединений между серверами Exchange 2013 и Office 365.
    
- Настройка записей DNS для указания необходимого маршрута входящих сообщений электронной почты.
    
#### <a name="exchange-server-on-premises"></a>Локальный сервер Exchange Server

- Настройка необходимых сертификатов для служб Exchange.
    
- Подготовка серверов к работе.
    
- Внедрение топологии маршрутизации сообщений Exchange.
    
- Внедрение групп обеспечения доступности баз данных.
    
- Обновление и обслуживание серверов Exchange.
    
- Добавление или отключение серверов в зависимости от нагрузки.
    
#### <a name="provider-hosted-exchange"></a>Exchange с размещением у поставщика

Поставщик отвечает за следующие функции:
  
- Системное и эксплуатационное обслуживание.
    
- Внедрение новых компонентов.
    
- Защита данных и аварийное восстановление.
    
В обязанности ИТ-специалистов вашей организации входит создание и контроль учетных записей пользователей.

## <a name="descriptions-of-three-new-or-updated-features-in-exchange-2013"></a>Описания трех новых или обновленных компонентов Exchange 2013

### <a name="exchange-online-protection"></a>Exchange Online Protection

Служба Exchange Online Protection (EOP) обеспечивает защиту любого варианта развертывания от спама и вредоносных программ при помощи набора защитных компонентов, которые внедряются в глобальной сети центров обработки данных. Это позволяет упростить контроль сред обмена сообщениями. Служба EOP включена в набор подписок Exchange Online, но ее также можно использовать для гибридного или локального развертывания.
  
На сопутствующих диаграммах представлены варианты развертывания для Exchange Online, гибридного развертывания Exchange и локальной среды Exchange, которые включают службу EOP в глобальной сети.
  
### <a name="exchange-server-deployment-assistant"></a>Помощник по развертыванию Exchange Server

Помощник по развертыванию Exchange Server — это веб-инструмент, который запрашивает у вас определенную информацию о существующей среде и поэтапно составляет индивидуальный контрольный список, чтобы облегчить развертывание Exchange Server для разных сценариев. Помощник по развертыванию Exchange Server составляет индивидуальный контрольный список развертывания именно для вашего сценария, будь то переход с более ранней версии Exchange на Exchange 2013, переход на Exchange Online или построение гибридной инфраструктуры.
  
На сопутствующем изображении представлен пример контрольного списка, составленного средствами Помощника по развертыванию Exchange Server.
  
### <a name="integration-with-lync-and-sharepoint"></a>Интеграция с Lync и SharePoint

В состав Exchange Server 2013 входит множество компонентов, интегрируемых с Lync Server 2013 и SharePoint Server 2013. В совокупности эти продукты располагают обширным набором функций и позволяют улучшить взаимодействие сотрудников внутри организации. 
  
На сопутствующей диаграмме представлен плакат по функциям проверки подлинности сервера и ссылка на него. 
  
- Архивация, хранение и обнаружение электронных данных
    
- Почтовые ящики сайта
    
- Единое хранилище контактов
    
- Фотографии пользователей в высоком разрешении
    
- Сведения о присутствии Lync в Outlook и Outlook Web App
    
- Проверка подлинности сервера
    
- Голосовая почта
    
- Записи собраний
    
- Синхронизация задач Exchange
    
На сопутствующей диаграмме представлен плакат, посвященный архитектуре Exchange Server 2013 с пакетом обновления 1 (SP1) и ссылка на него.
