# <a name="network-requests-in-office-2016-for-mac"></a>Сетевые запросы в Office 2016 для Mac

2016 Office для Mac приложений предоставляют возможности встроенное приложение на платформе macOS. Каждое приложение предназначена для работы в различных сценариях, включая состояния при доступности нет доступа к сети. Если компьютер подключен к сети, приложения автоматически подключаться к серии веб-службы предоставляют расширенные функциональные возможности. В этом документе описывается, какие конечные точки и URL-адресов приложений пытаться достигнуть и службы, предоставляемые. Эта информация полезна при устранении неполадок в конфигурации сети и настройке политик для сетевого прокси-серверы. Дополнительные сведения в этой статье предназначены для дополняют [статьи диапазонов URL-адреса Office 365 и адрес](urls-and-ip-address-ranges.md), который включает конечных точек для компьютеров под управлением Microsoft Windows.
  
Большая часть в этой статье — таблиц, о сети URL-адреса, тип и описание службы или функции, предоставляемые этой конечной точкой. Каждый из приложений Office, могут отличаться в его использования службы и конечной точки. В таблицах ниже определяются следующие приложения:
  
- W: Word
- P: PowerPoint
- X: Excel
- O: outlook
- N: OneNote
   
Тип URL-адреса определяется следующим образом:
  
- ST: Static - URL-адрес жестко задан в клиентском приложении.
    
- SS: Точками Static-URL-адрес кодируется как часть веб-страницы или перенаправитель.
    
- CS: Служба Config - URL-адрес, возвращается как часть служба настройки Office.

## <a name="options-for-reducing-network-requests-and-traffic"></a>Возможности сокращения трафика и сетевые запросы

Конфигурацию по умолчанию Office 2016 для Mac обеспечивает максимальное удобство работы пользователей, как функциональные возможности и сохранение этого компьютера в актуальном состоянии. В некоторых случаях может потребоваться от конечных точек сети общаться с приложениями. В этом разделе обсуждаются способам.
  
 ### <a name="disabling-cloud-sign-in-and-office-add-ins"></a>Отключение надстроек входа в облаке и Office
  
Пользователи корпоративных лицензий могут иметь строгие о сохранении документов для хранения на основе облака. Для отключения MSA/OrgID входа и доступ к надстроек Office можно задать следующие настройки каждого приложения.
  
- ```defaults write com.microsoft.Word UseOnlineContent -integer 0```

- ```defaults write com.microsoft.Excel UseOnlineContent -integer 0```

- ```defaults write com.microsoft.Powerpoint UseOnlineContent -integer 0```

При попытке доступа к функции входа в их появится сообщение об ошибке, что не в сети. Поскольку этот параметр также блокирует активации продуктов, он должен использоваться только для многократной установки. В частности с помощью этого режима запретить доступ к следующей конечные точки приложений Office:
  
- ```https://odc.officeapps.live.com```
    
- ```https://*.firstpartyapps.oaspapps.com```
    
- Все конечные точки, перечисленные в входа в предыдущем разделе.
    
- Все конечные точки, перечисленных в разделе «Смарт-подстановки» выше.
    
- Все конечные точки, перечисленные в предыдущем разделе «Активация продукта».
    
- Все конечные точки, перечисленные в предыдущем разделе «Приложения Office (также называемого add-ins)».
    
Для восстановления полной функциональности для пользователя, либо задайте установку на "2" или удалить его.
  
> [!NOTE]
> Этот параметр требует Office 2016 для Mac построения 15.25 [160726] или более поздняя версия. 
  
### <a name="telemetry"></a>Телеметрии
  
2016 Office для Mac отправляет данные телеметрии в корпорацию Майкрософт через регулярные интервалы времени. Данных передается в конечную точку «Хранилища». Данные телеметрии поможет команды разработчиков оценки работоспособности и непредвиденные поведение каждого приложения Office. Существует два типа данных телеметрии.
  
- **Периодический сигнал** содержит сведения о версии и лицензии. Эти данные отправляются сразу же после запуска приложения. 
    
- **Об использовании** содержит сведения об использовании приложений и некритических ошибок. Эти данные отправляются каждые 60 минут. 
    
Microsoft конфиденциальности очень серьезно. Сведения о политике сбора данных корпорации Майкрософт в [https://privacy.microsoft.com](https://privacy.microsoft.com). Чтобы запретить приложения отправляют телеметрии «Использование», можно настроить предпочтения **SendAllTelemetryEnabled** . Предпочтение каждого приложения, а можно задать с помощью macOS профили конфигурации или вручную с терминала: 
  
```defaults write com.microsoft.Word SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Excel SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Powerpoint SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Outlook SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.onenote.mac SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.autoupdate2 SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Office365ServiceV2 SendAllTelemetryEnabled -bool FALSE```

Периодический сигнал телеметрии всегда отправляется и не может быть отключен.
  
### <a name="crash-reporting"></a>Сбой отчетов
  
При возникновении ошибки сообщения о неустранимых приложения, приложение неожиданно прерывания и отправить отчет о сбоях в службу «Ватсон». Отчет о сбоях состоит из стека вызовов, представляющий список шагов обработки привели к сбою приложения. Следующие действия помогут определить точное функцию, завершившихся сбоем команды разработчиков и почему.
  
В некоторых случаях содержимое документа будет отображено происходит сбой приложения. Если приложение определяет документ как причину, он будет запрашивать у пользователя имеется также отправить документ, а также стек вызовов. Пользователи могут вносить принимать обоснованные решения на этот вопрос. ИТ-администраторы могут иметь строгие требования о передавать документы и принять решение от имени пользователя, никогда не отправлять документы. Чтобы предотвратить отправку документов и для отмены вывода строки для пользователя можно задать следующие настройки:
  
```defaults write com.microsoft.errorreporting IsAttachFilesEnabled -bool FALSE```

> [!NOTE]
> Если **SendAllTelemetryEnabled** имеет значение **FALSE**, все сбой отчетов о том, что процесс отключен. Чтобы включить сбой отчетов без отправки данных об использовании телеметрии, можно задать следующие настройки:```defaults write com.microsoft.errorreporting IsMerpEnabled -bool TRUE``` 
  
### <a name="updates"></a>Обновления
  
Корпорация Майкрософт выпускает Office 2016 для Mac обновлений через регулярные интервалы времени (обычно раз в месяц). Мы настоятельно рекомендуем пользователей и ИТ-администраторам обновлять машины для обеспечения последние исправления безопасности установлены. В тех случаях, где ИТ-администраторам требуется точно контролировать и управление обновлениями компьютера можно задать следующие настройки для предотвращения процесс AutoUpdate из автоматическое обнаружение и предложения обновлений продукта:
  
```defaults write com.microsoft.autoupdate2 HowToCheck -string 'Manual'```

### <a name="blocking-requests-with-a-firewallproxy"></a>Блокирование запросов с брандмауэр или прокси-сервера
  
Если вашей организации блоки запросы на URL-адреса через брандмауэр или прокси-сервер необходимо настроить URL-адреса, приведенные в этом документе как разрешенных или заблокировать из списка с 40 X ответа (например, 403 или 404). 40 X ответа будет позволяют приложениям Office соответствующим образом принимать отсутствия прав на доступ к ресурсу и будет предоставлять быстрее пользовательского интерфейса, чем просто удаление подключения, который в свою очередь приведет к клиенту выполнить повтор.
  
Если прокси-сервер требует проверки подлинности, будут возвращены 407 ответа клиенту. Для обеспечения наилучшего взаимодействия убедитесь, что вы используете Office 2016 сборки 15.27 или более поздней версии, как они содержат определенные исправления для работы с серверами NTLM и Kerberos.
