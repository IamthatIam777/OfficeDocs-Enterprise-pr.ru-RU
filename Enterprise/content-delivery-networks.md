## <a name="how-do-cdns-make-services-work-faster"></a>Каким образом сети доставки содержимого ускоряют работу служб?

Так как Office 365 использует архитектуру, которая включает в себя CDN, значки, сценарии и другой универсальный контент можно загрузить с серверов ближе к клиентским компьютерам, ускорение загружаемые файлы. Это означает, что более быстрый доступ к содержимому личные, которая безопасно хранится в Office 365 центров обработки данных.
  
## <a name="what-exactly-is-a-cdn"></a>Что такое CDN?

CDN используются большинство облачные службы enterprise. Облачных службах, например Office 365 имеют миллионам клиентов, одновременно загрузка сочетание собственный контент (например, по электронной почте) и универсальные контент (например, значки). Рекомендуется поместить изображения, которые использует всем пользователям, такими как значки, как можно ближе к на компьютере по мере возможности. Еще не применимы в каждом облачная служба для построения CDN центров обработки данных, который сохраняет этот универсальный контент в каждом общерегионального области или даже в каждый основной сервер-концентратор Интернета по всему миру, некоторые из этих CDN общих.
  
CDN может быть частный или общедоступный. Частный CDN принадлежат и обслуживается одним компании и только приложений и служб, компании могут использовать его. Общедоступный CDN выполняются теми компаниями, которые аренду об использовании для нескольких организаций. В зависимости от того, где вы находитесь может быть наиболее эффективным для Office 365 для загрузки универсальный изображений из CDN, который несет ответственность за Office 365 и потоками, общедоступных CDN или комбинацией этих двух. Независимо от того, какой тип CDN используется действия для получения данных, совпадают.
  
1. Клиент запрашивает данные от Office 365.

2. Office 365 возвращает данные вашего клиента или направляет клиента для CDN.

3. Если данные уже кэшируются на CDN, клиент файлы для загрузки данных непосредственно из ближайшего расположения CDN клиент в сети Интернет.

4. Если данные не кэшируются на CDN, узел CDN запрашивает данные из Office 365 и затем кэша 's данных в течение периода времени, после вашего клиента файлы для загрузки данных.

CDN по запросу файлов и изображений из ближайшего центра обработки данных Office 365 и в свою очередь, клиент запрашивает файлов и изображений из сети CDN ближайшего. Когда пользователи имеют доступ к облачной службе, как при чтении электронной почты в Outlook Web App в браузере пользователя пытается получить файлов и изображений из центра обработки данных Office 365. Вместо того чтобы тратить время и пропускной способности, передачей файлов, Office 365 перенаправляет браузер CDN. CDN решает, ближайший центр обработки данных в браузере и с помощью перенаправления, файлы для загрузки универсальный изображений из него. С помощью перенаправления CDN быстрого и позволяет пользователям много времени загрузки.

## <a name="is-there-a-list-of-all-the-cdns-that-office-365-uses"></a>Есть ли список всех CDN, которые использует Office 365?

CDN используется Office 365, всегда может быть изменен и во многих случаях существует несколько CDN партнерах, настроенных в том случае, если он недоступен. Два наиболее распространенных CDN используется являются [Akamai](https://www.akamai.com/us/en/cdn.jsp) и [Microsoft Azure](https://azure.microsoft.com/documentation/services/cdn/). Оба эти решения CDN имеют мире повышения доступности службы для нескольких углов мира. Контент, который используется для сохранения включает в себя общие сценарии Office 365, файлов и изображений. Например при входе в portal.office.com изображения берутся из ближайшего CDN, чтобы ускорить время загрузки страницы. Другие примеры Office 365 ProPlus хранения двоичных файлов установки на CDN для ускорения время, необходимое для загрузки последней версии Office. Имеется также некоторые конфиденциальным контента, хранящегося на CDN например видео файлы для Office 365 видео. Если вы загрузили видеоролики, файлы шифруются и затем сохраняются в своих зашифрованного сообщения со службами Azure мультимедиа. Когда видеопроигрыватель Office 365 извлекает видео сначала кэшируется к ближайшему CDN до загрузки для ускорения время, необходимое для загрузки видео.
