# <a name="using-bgp-communities-in-expressroute-for-office-365-scenarios"></a>Использование протокола BGP сообществ в ExpressRoute для сценариев Office 365

Подключение к Office 365 с помощью Azure ExpressRoute выполняется на основе протокола BGP рекламы конкретных IP-подсети, которые представляют сетей, где развернут конечных точках Office 365. Из-за универсальность Office 365 и число служб, которые составляют Office 365 клиенты могут часто потребность в управлении объявления, которое они принять во внутренней сети. Уменьшение числа IP-подсети; называется префиксы IP-адресов в оставшейся части этой статьи, в соответствии с терминология управления сетевого протокола BGP, служит следующие цели плана для клиентов:
  
- **Управление заявленным IP-адресов префиксы принятия** - клиенты, использующие инфраструктуру внутренней сети или оператору сети, который поддерживает только ограниченное количество префиксы IP-адресов и пользователям поставщика сети, плата за принятие префиксы выше ограниченного числа будет необходимо оценить общее число префиксы уже объявление для своей сети и выберите приложения Office 365, лучше всего подходит для ExpressRoute.

- **Управление пропускной способности на канал Azure ExpressRoute требуется** - клиенты могут воспользоваться контролировать конверт пропускной способности служб Office 365 ExpressRoute путь или пути в Интернете. Это позволяет зарезервировать ExpressRoute пропускной способности для конкретных приложений, таких как Скайп для бизнеса и маршрутизировать оставшиеся приложений Office 365 через Интернет путь.

Чтобы помочь пользователям этих целей, префиксы IP-адресов для Office 365, объявленные через ExpressRoute помеченных отдельные значения сообщества протокола BGP службы как показано в следующем примере.
  
> [!NOTE]
> Следует ожидать сетевой трафик, связанный с другими приложениями, должны быть включены в значение сообщества. Это ожидаемое поведение для глобального программного обеспечения как услугам с помощью общих служб и центров обработки данных. Это свернутый где это возможно с помощью выше две цели, управление count префикс и/или пропускной способности в виду.

|**Служба**|**Значение протокола BGP сообщества**|**Примечания**|
|:-----|:-----|:-----|
|Exchange\*  <br/> |12076:5010  <br/> |Включает в себя службы Exchange и EOP\*  <br/> |
|SharePoint\*  <br/> |12076:5020  <br/> |SharePoint Online  <br/> |
|Скайп для бизнеса\*  <br/> |12076:5030  <br/> |Skype для бизнеса Online  <br/> |
|другие службы Office 365\*  <br/> |12076:5100  <br/> |Включает в себя Azure Active Directory (сценарии проверки подлинности и синхронизации службы каталогов) и services портала Office 365  <br/> |
|\*Область сценария службы, включенные в ExpressRoute описана в статье [конечных точках Office 365](https://aka.ms/o365endpoints) .  <br/> \*\*Дополнительные службы и протокола BGP сообщества значения могут быть добавлены в будущем. [Текущий список протокола BGP сообщества](https://azure.microsoft.com/documentation/articles/expressroute-routing/).<br/> |

## <a name="what-are-the-most-common-scenarios-for-using-bgp-communities"></a>Что такое наиболее распространенные сценарии для использования протокола BGP сообществ (en)

Клиенты могут использовать протокола BGP сообществ для регулирования групп префиксы IP-адресов, которые были приняты свои сети с помощью ExpressRoute Azure, таким образом влияние на общее число префикса IP-адресов и конверт ожидаемый пропускной способности некоторых служб Office 365. Важно понимать, что все Office 365 потребует привязанное трафика вне зависимости от использования Azure ExpressRoute или протокола BGP сообщества Интернета. Следующие три сценария являются наиболее распространенные способы использования этой функции.
  
### <a name="scenario-1-minimizing-the-number-of-office-365-ip-prefixes"></a>Сценарий 1: Минимизации количества префиксы IP-адресов для Office 365

Корпорации Contoso — это организация 50 000 человека, в настоящее время Exchange Online и SharePoint Online с помощью Office 365. При изучении ExpressRoute требования к Contoso определяет его сетевых устройств в разных местах региональных не может обрабатывать маршрутизации в таблице размером более 100 записей дополнительные маршрут. Contoso просмотреть общее число префиксы IP-адресов, которые ExpressRoute бы advertise полный набор служб Office 365 и предполагает, что оно превышает 100. Чтобы превышать 100 записей дополнительные маршрут, Contoso охватывает использование ExpressRoute для Office 365 значение только по SharePoint Online протокола BGP сообщества, 12076:5020, получаемую через ExpressRoute Microsoft авторами.

|**Тег сообщества протокола BGP используется**|**Функциональные возможности маршрутизируемые через Azure ExpressRoute**|**Требуется Internet маршрутов**|
|:-----|:-----|:-----|
|SharePoint  <br/> (12076:5020)  <br/> |SharePoint Online &amp; OneDrive для бизнеса  <br/> | DNS, списки отзыва Сертификатов, &amp; CDN запросов  <br/>  Все другие службы Office 365, специально не поддерживается по Azure ExpressRoute  <br/>  Все прочие облачные службы Майкрософт  <br/>  Портал Office 365, проверки подлинности в Office 365, &amp; Office Online  <br/>  Exchange Online, Exchange Online Protection и Скайп для бизнеса Интернет-версия  <br/> |

> [!NOTE]
> Для достижения снижение числа префикс для каждой службы, будет сохраняться минимальный объем перекрытие между службами. Это ожидаемое поведение.
  
### <a name="scenario-2-scoping-expressroute-and-internal-bandwidth-use-to-some-office-365-services"></a>Сценарий 2: Области видимости ExpressRoute и внутреннюю полосу пропускания используют для некоторых служб Office 365

Fabrikam Inc, больших глобальной экономике enterprise с распределенной сети разнородных является подписчиком множество служб Office 365, включая; Exchange Online, SharePoint Online и Скайп для бизнеса в Интернете. Внутренний маршрутизации инфраструктуры компании Fabrikam может обрабатывать тысячи префиксы IP-адресов в его таблицы маршрутизации; Тем не менее Fabrikam требуется только для подготовки ExpressRoute и внутреннюю полосу пропускания для приложений Office 365, которые являются конфиденциальными производительность сети и использовать их существующих пропускной способности сети для всех других приложений Office 365.
  
По этой причине Fabrikam областей действия ее Azure ExpressRoute пропускной способности для только что Скайп для сетевого протокола BGP сообщества Business значения 12076:5030, получаемую через ExpressRoute Microsoft авторами. Оставшиеся сетевой трафик, связанный с Office 365 продолжает использовать точек исходящего трафика Интернета.

|**Тег сообщества протокола BGP используется**|**Функциональные возможности маршрутизируемые через Azure ExpressRoute**|**Требуется Internet маршрутов**|
|:-----|:-----|:-----|
|Skype для бизнеса  <br/> (12076:5030)  <br/> |Скайп SIP-сигналы, файлы для загрузки, общий доступ к голосовой связи, видео и рабочего стола  <br/> | DNS, списки отзыва Сертификатов, &amp; CDN запросов  <br/>  Все другие службы Office 365, специально не поддерживается по Azure ExpressRoute  <br/>  Все прочие облачные службы Майкрософт  <br/>  Портал Office 365, проверки подлинности в Office 365, &amp; Office Online  <br/>  Скайп для бизнеса телеметрии, Скайп клиента Краткие советы, общедоступной службы обмена Мгновенными сообщениями  <br/>  Exchange Online, Exchange Online Protection и SharePoint Online  <br/> |

### <a name="scenario-3-scoping-azure-expressroute-for-office-365-services-only"></a>Сценарий 3: Областей Azure ExpressRoute для служб Office 365

Банка Woodgrove — это клиент несколько облачные службы Майкрософт, включая Office 365. После оценки своей сети емкости и потребление банка Woodgrove решает развернуть Azure ExpressRoute в качестве предпочитаемый путь для поддерживаемых служб Office 365. Таблицы маршрутизации может поддерживать полный набор префиксы IP-адресов для Office 365 и цепи Azure ExpressRoute, которые они предоставил поддерживают все планируемый пропускная способность и задержка потребности.
  
Для обеспечения сетевого трафика, связанного с помощью служб Microsoft cloud, отличное от Office 365, банка Woodgrove охватывает использование ExpressRoute для Office 365 для всех префиксов IP-адресов с меткой с Office 365 конкретного протокола BGP сообщества значения, 12076:5010, 12076:5020, 12076:5030, 12076:5100.

## <a name="key-planning-considerations-to-using-bgp-communities"></a>Основные сведения о к использованию протокола BGP сообществ планировании

Пользователи, чтобы воспользоваться преимуществами протокола BGP сообщества, влияющих на способ ExpressRoute объявление и распространения через сеть клиента должен выполнить следующие условия:
  
- При использовании протокола BGP сообществ в проекта сети, он очень важен для обеспечения симметричный маршрута по-прежнему поддерживается. В некоторых случаях добавление или удаление протокола BGP сообществ может создавать ситуации, где симметричного маршрутизации будет отключена и конфигурацию маршрутизации необходимо обновить, чтобы восстановить симметричный маршрутизации.

- Области Azure ExpressRoute с помощью протокола BGP сообщества значения — это действие клиента. Корпорация Майкрософт будет advertise всеми префиксами IP-адрес, связанный с авторами связь вне зависимости от области видимости, настроенных с клиентом.

- Azure ExpressRoute не поддерживает все действия в зависимости от клиента, назначенный сообществ протокола BGP сети корпорации Майкрософт.

- Префиксы IP-адрес, используемый Office 365are только помеченные значениями службы конкретного протокола BGP сообщества, расположение, которое конкретного протокола BGP сообществ не поддерживаются. Службы Office 365 являются глобальными характер и фильтрации на основе местоположения клиента префиксами или данных в облаке Office 365 не поддерживается. Рекомендуется настроить для координации Минимальная или наиболее предпочтительную сетевой путь из сетевой папки пользователя в глобальной сети Microsoft, вне зависимости от физического расположения IP-адрес службы Office 365 запрашивается.

- Префиксы IP-адресов, включенных в каждом значении сообщества протокола BGP представляют подсети, которая содержит IP-адреса для приложения Office 365, связанные со значением. В некоторых случаях более одного приложения Office 365 имеет IP-адресов в подсети, приведшего к префикс IP-адресов, существующими в более одного значения сообщества. Это является ошибкой, хотя редко, из-за распределения фрагментации и не влияет на цели управления префикс счетчик или пропускной способности. Пользователям, рекомендуется использовать «разрешить необходимые» подход отличие от «запретить не требуется, что» при использование преимуществ протокола BGP сообщества Office 365 для сведения к минимуму влияние.

- С помощью протокола BGP сообществ не изменяется базовой требования сетевого подключения или конфигурации, необходимые для использования Office 365. Пользователи, для доступа к Office 365, по-прежнему требуется, чтобы иметь возможность подключения к Интернету.

- Области Azure ExpressRoute с помощью протокола BGP сообщества влияет только на маршрутов, которые внутренней сети могут видеть по связям авторами Microsoft. Может потребоваться выполнить дополнительные приложения уровня настройки таких как использование PAC или WPAD конфигурации в сочетании с заданной областью маршрутизации.

- С помощью Microsoft, назначенные протокола BGP сообщества, клиенты могут использовать для назначения собственных сообществах протокола BGP получены с помощью Azure ExpressRoute влияния внутренней маршрутизации на префиксы IP-адресов для Office 365. Пример популярные использования назначение сообщества расположение на основе протокола BGP все маршруты, полученные по каждой указанной ExpressRoute авторами расположение и затем использовать эти сведения более низком уровне в сети клиента для координации Минимальная или самый предпочитаемый путь сети в Сеть корпорации Майкрософт. Клиентов, назначенных протокола BGP сообщества с ExpressRoute для Office 365 сценариев используется вне области действия элемента управления Microsoft или видимости.
