# <a name="use-the-page-diagnostics-tool-for-sharepoint-online"></a>Использование средства диагностики страницы для SharePoint Online

В этой статье описывается, как использовать средство диагностики страницы для анализа классический публикации страниц и страниц веб-сайтов классический групп, от подмножество рекомендации в **SharePoint Online**. 
  
Сайты групп, имеющих публикации включен нельзя использовать CDN, но все остальные правила могут быть применены. Публикация требует дополнительных затрат, поэтому не включать публикации для получения CDN функциональные возможности, как оно может отрицательно сказаться на время загрузки страницы.
  
> [!IMPORTANT]
> Средство диагностики страница будет не выполняться библиотеки документов или страниц системы, как это средство предназначено для просмотра страниц сайта SharePoint. На странице *allitems.aspx* представляет собой страницу системы. Если попытаться запустить средство на странице системы, вы получите сообщение, «это приложение должно выполняться только на страницах SharePoint.» > ![необходимо запустить на странице SharePoint](media/34aadfff-1009-496b-9c87-4fc2780e017c.png)
  
> Это не ошибка в средстве как не имеет смысла при оценке библиотеки или страницы системы. Перейдите на страницу SharePoint не системы по использованию средства. Следует вы хотите отправить отзыв о средство Пожалуйста нажмите кнопку «о программе» и следуйте [предоставить ссылку свои отзывы и предложения](https://go.microsoft.com/fwlink/?linkid=874109).

1. Откройте с помощью браузера Chrome, [ссылка на средство](https://chrome.google.com/webstore/detail/inahogkhlkbkjkkaleonemeijihmfagi) напрямую или откройте поиска в [Веб-браузер Chrome хранилища](https://chrome.google.com/webstore/search/page%20diagnostics%20for%20sharepoint) и установите расширение браузера.

Это означает, что средство считывает запросы и ответы при нажатии кнопки Пуск и только для активной вкладкой SharePoint установленным средство. Эти сведения регистрируются локально веб-обозревателем и доступен через Экспорт в JSON ссылок в средстве. **Сведения не отправляется или записанного с Microsoft.**

Только данные, записываемые средство — это имя клиента, правила число и ли вариант поддержки ведения журнала было используются при запуске средства. Эти сведения — корпорации Майкрософт для анализа, какие проблемы, связанные с возникающие с наших клиентов и убедитесь, что функция ведения журнала поддержка — это не использования не по назначению.
  
Средство учитывает конфиденциальности Microsoft политики доступны [здесь](https://go.microsoft.com/fwlink/p/?linkid=857875).

Please follow your Company's own Privacy guidelines before sharing the JSON file outside of your Company as it contains the URL's and they fall within PII (Personally Identifiable Information).

Прежде чем нажать кнопку **Остановить**, убедитесь в том, что данные трассировки сети завершен. В противном случае будут иметь частичное трассировки. 
  
Кроме того средство — это расширение браузера и открытие нескольких вкладок или windows допускает только один активный экземпляр tool, чтобы запустить одновременно. Это ограничение расширений в браузере. 
  
4. Щелкните на эмблема. ![Страница диагностики для логотип SharePoint](media/60a3e44d-1b59-483f-b50f-d580044d921a.png) Чтобы загрузить средство и вам предоставят всплывающего окна следующие расширения:

5. Первую ссылку связь **о** будет предоставлять общие рекомендации и Дополнительные сведения об средство, включая ссылку пользователю в этой статье. Он также включает прямая ссылка на рекомендации по производительности SharePoint, уведомление сторонних производителей и параметр, чтобы оставить отзыв о средстве. 
    
6. Изучите раздел **идентификатор корреляции, SPRequestDuration, SPIISLatency** **, страница загружена времени и URL-адрес** . (В этом разделе информационные и может использоваться для нескольких целей). 
    
  - **CorrelationID** является важным элементом, при работе с группы поддержки Microsoft, как она позволит обращаться к дополнительные диагностические данные по запросу. 
    
  - **SPRequestDuration** — это сервер время обработки страницы. Если в данный момент времени, она не обязательно означает, что сервер был неправильно для выполнения, но можно также числа звонков и нагрузки на компьютерах с этой страницы на сервере например структуры навигации, большие изображения, большое количество вызовов API может учитываться в значительно больше времени сервера . 
    
  - **SPIISLatency** — это время в миллисекундах, в случае веб-сервера переднего плана, если она получает запрос на загрузку страницы. Это указывает задержку для запуска обработки страницы и не содержит время, затрачиваемое для веб-приложения для ответа. 
    
  - **Время загрузки страницы** — это время, записанные с помощью страницы с момента запроса во время ответа получено и чтение в окне обозревателя. Любое дополнительное время зависит от производительности компьютера, и время, необходимое для браузера для загрузки. 
    
  - **URL-адрес** (URL-адрес) — это веб-адрес текущей страницы. 
    
7. **Диагностики вкладки** будут перечислены правила и ли какие-либо их отмечены красным ![кросс-](media/9859ac84-be43-4eae-984c-e0e827f5a228.png), а затем проблем, обнаруженных на странице.
    
    Каждое правило имеет свой собственный ссылку «Дополнительно», если нажать кнопку на его, когда он будет красным. Вы перейдете к подробные сведения о правила и способы устранения проблемы.
    
    ![Диагностика красный - правило open](media/1598f0f7-3103-4613-8787-dfec6fffd40a.png)
  
1. Проверка работает как обычного пользователя
  
Проверка производительности страниц не следует применять при входе в качестве учетной записи службы, администратор или администратор семейства веб-сайтов т.е. учетную запись с повышенными привилегиями. Дополнительные сценарии и функциональные возможности загружается специально для этих типов учетных записей и не будет предоставлять значение true, представление производительности страниц.
    
2. Проверка запросов в SharePoint
  
Объем данных и запросы, выполненные на сервере должны быть ограничены, как перегруженные страница будет сталкиваются с низким быстродействием. Эта проверка проверяет число запросов в SharePoint и будут выводиться рекомендации при запросах превышает 6 запросов. Большинство запросов должен быть кэширования и поэтому не вызывается при каждой загрузке страницы. Кэш должны быть установлены и использовать по крайней мере 15 минут для сокращения объема звонков на страницу с каждого пользователя. Это распространенные проблемы и в большинстве случаев только изменения данных ежедневно Однако страницы проверяет и получает данные каждый раз для каждой страницы для каждого пользователя, который часто не требуется.
    
3. Проверьте использование CDN
  
Контента сети доставки предоставила корпорации Майкрософт и из них, делается ссылка, вот сети доставки Online контента SharePoint. Существует несколько типов и различных службах CDN, например CDN SharePoint, а затем CDN в Azure. [Используйте следующие рекомендации](https://go.microsoft.com/fwlink/?linkid=873250).
    
4. Проверка размеров большие изображения
  
Изображения следует оптимизировать для веб-благодаря использованию более web типов, таких как PNG. Показ изображений также должны использоваться и напрямую доступно на сайте SharePoint. Изображения аудио- и изображений, размер которых превышает 100 КБ, будут отмечены как не оптимизирована для веб-сайта. [Используйте следующие рекомендации для оптимизации изображений](https://go.microsoft.com/fwlink/?linkid=873251).
    
5. Проверка структуры навигации
  
Структуры навигации первоначально был разработан для использования в SharePoint в локальной которых может использоваться кэша объектов. Структуры навигации не рекомендуется для использования в SharePoint Online и должно быть изменено на управляемой навигации или настраиваемый поставщик. [, Используйте следующие рекомендации по оптимизации перехода.](https://go.microsoft.com/fwlink/?linkid=873247)
    
6. Проверка веб-части CBQ (CBQ - контента по веб-части запроса)
  
Контент по веб-части запроса создает высокой нагрузки SQL, как он проходит все элементы в запрос для каждой загрузки страницы для каждого пользователя. В отличие от локальной установки нет не кэша, можно ограничить число запросов, необходимые для заполнения этой веб-части. Таким образом CBQ работает медленно и снижает общую производительность страницы, поэтому он не должен использоваться. Используйте замены для контента веб-части запроса содержимого веб-части поиска (CSWP). [Используйте следующие рекомендации, относящиеся к веб-часть поиска контента](https://go.microsoft.com/fwlink/?linkid=873245).
    
8. **Вкладка данных трассировки сетевого** предоставляет *** подробные сведения о запросах для создания страницы, а также полученных ответов. Производительность каждого запроса и ответа используется цветовое на основании их влияние на общую производительность страницы т.е. Зеленая \< мс, желтая 1000 500 мс и красный \> 1000 мс. 
    
    На этой вкладке — Экспорт в параметр JSON следует нужно загрузить и совместное использование сведений запроса и ответа.
    
    > [!IMPORTANT]
    > Наведите указатель мыши на сокращенный URL-адрес для просмотра полный URL-адрес. 
  
    ![Трассировка сети](media/3cfede99-7d31-4041-888d-7bbc275cadc2.png)
  
    На рисунке красный значок является самой страницы и, которая всегда будет отображаться красный Если страница загружается в \< 1000 мс (то есть 1 секунда).
    
    В некоторых случаях *будет нет времени или цвет индикатора, так как элементы уже были кэшированы браузера* . Чтобы проверить это правильно, откройте страницу, очистите кэш браузера и нажмите кнопку **Пуск** , которая отвечает за выполнение принудительной загрузки «холодного» страницы и иметь значение true, посредством загрузки начальной страницы. Это следует сравнивать для загрузки страницы «горячего» как, которое поможет определить, какие элементы кэшируются на странице. 
    
9. Если вы хотите предоставить разработчикам или пользователю эти сведения или сведения о нажмите кнопку «Экспорт для JSON» на приведенном выше рисунке и, который будет загрузить результаты. Обратите внимание, что файл можно затем открыть с помощью средства просмотра файлов JSON.
    
    > [!IMPORTANT]
    > Эти результаты будут содержать URL-адрес, поэтому он содержит PII (личные сведения) и требованиями к компании необходимо следовать перед отправкой их. 
  
10. Мы были включены **уровень поддержки Microsoft компонента** , который должен использоваться только при работе непосредственно для случая поддержки для производительности. Использование этой функции предоставления не дает преимущества при использовании без наша служба поддержки. На самом деле сделает странице значительно снижение производительности и продолжительное использование функции может рассматриваться как «неправильное» службы. Нет дополнительных сведений при использовании этой функции в средстве ведения журнала в службе при добавлении дополнительных сведений. 
    
    Никаких изменений не отображается, за исключением того, что вы будете получать уведомление, что вы включили и повысить производительность страницы будет значительно снизить 2 - 3 раза производительность может снизиться, пока, который включен. Он будет только важные для конкретной страницы и активного сеанса. По этой причине это следует применять и только когда активно участие наша служба поддержки.
    
    Чтобы включить функцию откройте средство и использовать сочетание клавиш ALT-Shift-L которого будет выведено «включить ведение журнала уровень поддержки». Щелкните флажок и затем нажмите кнопку запустить для повторной загрузки страницы и создания ведение подробного журнала для поддержки для анализа.
    
    ![Включить параметр поддержки](media/ddef47de-8593-4b28-9346-eb48ebf6cdab.png)
  
    Важные элемент базы данных для — CorrelationID как группа поддержки затем будет использовать этот номер, чтобы извлечь сведения, необходимые. Скопируйте CorrelationID и предоставляют, поддержка, как они не могут выполнять требуемые задачи полный код.
