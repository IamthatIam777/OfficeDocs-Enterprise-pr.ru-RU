# <a name="use-the-page-diagnostics-tool-for-sharepoint-online"></a>Использование средства диагностики страницы для SharePoint Online

В этой статье описывается, как использовать средство диагностики страницы для анализа классический публикации страниц и страниц веб-сайтов классический групп, от подмножество рекомендации в **SharePoint Online**. 
  
Сайты групп, имеющих публикации включен нельзя использовать CDN, но все остальные правила могут быть применены. Публикация требует дополнительных затрат, поэтому не включать публикации для получения CDN функциональные возможности, как оно может отрицательно сказаться на время загрузки страницы.
  
> [!IMPORTANT]
> Средство диагностики страница будет не выполняться библиотеки документов или страниц системы, как это средство предназначено для просмотра страниц сайта SharePoint. На странице *allitems.aspx* представляет собой страницу системы. Если попытаться запустить средство на странице системы, вы получите сообщение, «это приложение должно выполняться на страницах SharePoint.»<br/> ![Необходимо выполнить на странице SharePoint](media/34aadfff-1009-496b-9c87-4fc2780e017c.png)<br/>Это не ошибка в средстве как не имеет смысла при оценке библиотеки или страницы системы. Перейдите на страницу SharePoint не системы по использованию средства. Следует вы хотите отправить отзыв о средство Пожалуйста нажмите кнопку «о программе» и следуйте [предоставить ссылку свои отзывы и предложения](https://go.microsoft.com/fwlink/?linkid=874109). 
  
## <a name="install-the-page-diagnostic-tool"></a>Установите средство диагностики страницы

> [!IMPORTANT]
> Корпорация Майкрософт не поддерживает данные или посещаемых и мы не записывать любые личные сведения, веб-сайт или файл для загрузки данных с помощью этой программы. Только данные, записываемые средство — это имя клиента, правила число и ли вариант поддержки ведения журнала было используются при запуске средства. Эти сведения — корпорации Майкрософт для анализа, какие проблемы, связанные с возникающие с наших клиентов и убедитесь, что функция ведения журнала поддержка — это не использования не по назначению.

1. Откройте с помощью браузера Chrome, [ссылка на средство](https://chrome.google.com/webstore/detail/inahogkhlkbkjkkaleonemeijihmfagi) напрямую или откройте поиска в [Веб-браузер Chrome хранилища](https://chrome.google.com/webstore/search/page%20diagnostics%20for%20sharepoint) и установите расширение браузера. Просмотрите политику конфиденциальности пользователей, содержащимся на странице "Описание" в хранилище. При добавлении средство в браузере, вы увидите следующее Обратите внимание на то разрешения.<br/>![Разрешений для хранилища Chrome](media/e9fbcef0-1171-43ac-8ea8-c2b5be1b7925.png)<br/>   Это примечание будет готова, так как страница может содержать контент из расположения за пределами SharePoint, в зависимости от веб-части и настроек на странице. Это означает, что средство считывает запросы и ответы при нажатии кнопки Пуск и только для активной вкладкой SharePoint установленным средство. Эти сведения регистрируются локально веб-обозревателем и доступен через Экспорт в JSON ссылок в средстве. **Сведения не отправляется или записанного с Microsoft.** (Средство учитывает ли это Майкрософт о конфиденциальности политики доступны [здесь](https://go.microsoft.com/fwlink/p/?linkid=857875).)<br/><br/>Функциональные возможности «Экспорт для JSON» в средстве также является почему требуется разрешение «Управление загруженных файлов». Выполните инструкции конфиденциальности вашей компании перед общего доступа к файлу JSON за пределами вашей организации, как результаты, содержащие URL-адреса, который можно рассматривать как PII (личные сведения).
    
2. (Необязательно) Если вы хотите использовать средство в режиме incognito хрома, перейдите к данное расширение имени файла и нажмите кнопку **Разрешить в incognito**.
    
3. Перейдите к странице классический публикации SharePoint в SharePoint Online, который вы хотите просмотреть. Мы разрешены для «с задержкой» элементов на страницы; Таким образом, **средство не прекращается автоматически**. Следует требуется остановить семейства сайтов, можно нажать кнопку **Остановить**. (Это предусмотрено при проектировании для обслуживания для всех сценариев загрузки страниц.) Прежде чем нажать кнопку **Остановить**, убедитесь в том, что данные трассировки сети завершен. В противном случае будут иметь частичное трассировки. Кроме того средство — это расширение браузера и открытие нескольких вкладок или windows допускает только один активный экземпляр tool, чтобы запустить одновременно. Это ограничение расширений в браузере. 
  
4. Щелкните на эмблема. ![Страница диагностики для логотип SharePoint](media/60a3e44d-1b59-483f-b50f-d580044d921a.png) Чтобы загрузить средство и вам предоставят всплывающего окна следующие расширения:<br/> ![Средство диагностики страницы контекстного меню](media/b01fa00e-c5f3-4c37-91f2-6edd096cf87e.png)<br/>Запуск и остановка следить за операции начнется основные концепции при нажатии кнопки start, будет перезагрузка страницы и семейства сайтов.

Ознакомьтесь со следующими разделами, чтобы узнать больше о сведения, содержащиеся в средстве.

## <a name="what-youll-see-in-the-page-diagnostics-tool"></a>Что вы увидите в средство диагностики страницы
    
1. Ссылка **о** будет предоставлять общие рекомендации и Дополнительные сведения об средство, включая ссылку обратно в этой статье. Он также включает прямая ссылка на рекомендации по производительности SharePoint, уведомление сторонних производителей и параметр, чтобы оставить отзыв о средстве. 
    
2. **Идентификатор корреляции, SPRequestDuration, SPIISLatency**, **время загрузки страницы**и **URL-адрес** сведений являются информационными и может использоваться для нескольких целей. 
    
  - **CorrelationID** является важным элементом, при работе с группы поддержки Microsoft, как она позволит обращаться к дополнительные диагностические данные по запросу. 
    
  - **SPRequestDuration** — это сервер время обработки страницы. Если в данный момент времени, она не обязательно означает, что сервер был неправильно для выполнения, но можно также числа звонков и нагрузки на компьютерах с этой страницы на сервере например структуры навигации, большие изображения, большое количество вызовов API может учитываться в значительно больше времени сервера . 
    
  - **SPIISLatency** — это время в миллисекундах, в случае веб-сервера переднего плана, если она получает запрос на загрузку страницы. Это указывает задержку для запуска обработки страницы и не содержит время, затрачиваемое для веб-приложения для ответа. 
    
  - **Время загрузки страницы** — это время, записанные с помощью страницы с момента запроса во время ответа получено и чтение в окне обозревателя. Любое дополнительное время зависит от производительности компьютера, и время, необходимое для браузера для загрузки. 
    
  - **URL-адрес** (URL-адрес) — это веб-адрес текущей страницы. 
    
3. [ **Диагностики** вкладки](#how-to-use-the-diagnostic-tab) будут перечислены правила и ли какие-либо их отмечены красным ![кросс-](media/9859ac84-be43-4eae-984c-e0e827f5a228.png), а затем проблем, обнаруженных на странице.<br/>Каждое правило имеет свой собственный ссылку «Дополнительные сведения», щелкните Если элемент имеет красный цвет. Вы перейдете к подробные сведения о правила и способы устранения проблемы.<br/>![Диагностика красный - правило open](media/1598f0f7-3103-4613-8787-dfec6fffd40a.png)

4. [Вкладка **данных трассировки сетевого** ](#how-to-use-the-network-trace-tab) предоставляет подробные сведения о странице построение запросов и ответов.

## <a name="how-to-use-the-diagnostic-tab"></a>Как использовать вкладку диагностики

1. **Проверка, запущено как обычного пользователя**  Проверка производительности страниц не следует применять при входе в качестве учетной записи службы, администратор или администратор семейства веб-сайтов или любой учетной записи с повышенными привилегиями. Дополнительные сценарии и функциональные возможности загружаются специально для этих типов учетных записей, поэтому результаты не будет значение true, представление производительности страниц.
    
2. **Проверка запросов в SharePoint**  Объем данных и запросы, выполненные на сервере должны быть ограничены, как перегруженные страница будет сталкиваются с низким быстродействием. Эта проверка проверяет число запросов в SharePoint и будут выводиться рекомендации при запросах превышает 6 запросов. Большинство запросов должен быть кэширования и поэтому не вызывается при каждой загрузке страницы. Кэш должны быть установлены и использовать по крайней мере 15 минут для сокращения объема звонков на страницу с каждого пользователя. Это распространенные проблемы и в большинстве случаев только изменения данных ежедневно Однако страницы проверяет и получает данные каждый раз для каждой страницы для каждого пользователя, который часто не требуется.
    
3. **Проверьте использование CDN**  Контента сети доставки (CDN) было предоставлено корпорацией Майкрософт и из них, делается ссылка, вот сети доставки Online контента SharePoint. Существует несколько типов и различных службах CDN, например CDN SharePoint, а затем CDN в Azure. [Используйте следующие рекомендации](https://go.microsoft.com/fwlink/?linkid=873250).
    
4. **Поиск размеров большие изображения**  Изображения следует оптимизировать для веб-благодаря использованию более web типов, таких как PNG. Показ изображений также должны использоваться и напрямую доступно на сайте SharePoint. Изображения аудио- и изображений, размер которых превышает 100 КБ, будут отмечены как не оптимизирована для веб-сайта. [Используйте следующие рекомендации для оптимизации изображений](https://go.microsoft.com/fwlink/?linkid=873251).
    
5. **Проверка структуры навигации**  Структуры навигации первоначально был разработан для использования в SharePoint в локальной которых может использоваться кэша объектов. Структуры навигации не рекомендуется для использования в SharePoint Online и должно быть изменено на управляемой навигации или настраиваемый поставщик. [, Используйте следующие рекомендации по оптимизации перехода.](https://go.microsoft.com/fwlink/?linkid=873247)
    
6. **Проверить наличие CBQ веб-части** (CBQ - контента по веб-части запроса)  Контент по веб-части запроса создает высокой нагрузки SQL, как он проходит все элементы в запрос для каждой загрузки страницы для каждого пользователя. В отличие от локальной установки нет не кэша, можно ограничить число запросов, необходимые для заполнения этой веб-части. Таким образом CBQ работает медленно и снижает общую производительность страницы, поэтому он не должен использоваться. Используйте замены для контента веб-части запроса содержимого веб-части поиска (CSWP). [Используйте следующие рекомендации, относящиеся к веб-часть поиска контента](https://go.microsoft.com/fwlink/?linkid=873245).

## <a name="how-to-use-the-network-trace-tab"></a>Как использовать вкладку трассировки сети
    
На вкладке **Данных трассировки сетевого** предоставляет подробные сведения о запросах для создания страницы, а также полученных ответов. 

1. **Найдите время загрузки элементов, отмеченных как красный**. Производительность каждого запроса и ответа используется цветовое, основании их влияние на общую производительность страницы следующим образом:
- Зеленый: \< мс
- Желтый: 500-1000 мс
- Красный: \> 1000 мс
<br/>![Трассировка сети](media/3cfede99-7d31-4041-888d-7bbc275cadc2.png)<br/>На рисунке показано выше красной элемента относится к странице по умолчанию. Будут всегда отображать красного, если страница загружается в \< 1000 мс (менее 1 секунды).

2. **Время загрузки элемент теста**. В некоторых случаях будет нет времени или цвет индикатора, так как элементы уже были кэшированы браузера. Чтобы проверить это правильно, откройте страницу, очистите кэш браузера и нажмите кнопку **Пуск** , которая отвечает за выполнение принудительной загрузки «холодного» страницы и иметь значение true, посредством загрузки начальной страницы. Это следует сравнивать для загрузки страницы «горячего» как, которое поможет определить, какие элементы кэшируются на странице. 
    
3. **Полезные сведения совместный доступ с другими пользователями, которые могут помочь изучения проблем**. Для совместного использования сведений и сведения, содержащиеся в средстве с разработчиков или технического специалиста, нажмите кнопку **Экспорт в JSON** (как показано на рисунке выше). Который позволит загрузить результаты, доступных для просмотра с помощью средства просмотра файлов JSON.

> [!IMPORTANT]
> Эти результаты содержат URL-адреса и, который можно рассматривать как PII (личные сведения). Убедитесь, что следовать требованиям вашей организации перед отправкой их. 

## <a name="engaging-with-microsoft-support"></a>Включение поддержки Майкрософт
   
Мы были включены **уровень поддержки Microsoft компонента** , который должен использоваться только при работе непосредственно для случая поддержки для производительности. Использование этой функции предоставления не дает преимущества при использовании без наша служба поддержки. На самом деле сделает странице значительно снижение производительности и продолжительное использование функции может рассматриваться как «неправильное» службы. Нет дополнительных сведений при использовании этой функции в средстве ведения журнала в службе при добавлении дополнительных сведений. 

Никаких изменений не отображается, за исключением того, что вы будете получать уведомление, что вы включили и повысить производительность страницы будет значительно снизить 2 - 3 раза производительность может снизиться, пока, который включен. Он будет только важные для конкретной страницы и активного сеанса. По этой причине это следует применять и только когда активно участие наша служба поддержки.

### <a name="to-enable-the-microsoft-support-level-feature"></a>Чтобы включить функцию уровень поддержки Майкрософт

1. Откройте средство диагностики страницы.
2. На клавиатуре нажмите клавишу ALT, Shift, L. При этом отобразится **Ведение журнала в уровень поддержки**. 
3. Установите флажок и нажмите кнопку **запустить** для повторной загрузки страницы и создания подробное ведение журнала для поддержки для анализа.<br/>![Включить параметр поддержки](media/ddef47de-8593-4b28-9346-eb48ebf6cdab.png)
  
Важные элемент базы данных для — CorrelationID как группа поддержки затем будет использовать этот номер, чтобы извлечь сведения, необходимые. Скопируйте CorrelationID (в верхней части страницы диагностики) и предоставляют, поддержка, как они не могут выполнять требуемые задачи полный код.
