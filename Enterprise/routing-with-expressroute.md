# <a name="routing-with-expressroute-for-office-365"></a>Маршрутизация с помощью ExpressRoute для Office 365

Для правильного понимания трафик Office 365 с помощью Azure ExpressRoute, вам потребуются утвержденного может воспользоваться основные [требования к маршрутизации ExpressRoute](https://azure.microsoft.com/documentation/articles/expressroute-routing/) и [ExpressRoute цепи и маршрутизации доменов](https://azure.microsoft.com/documentation/articles/expressroute-circuit-peerings/). Эти размещения основах с помощью ExpressRoute, Office 365 клиенты будут основываться на записях.
  
Ниже перечислены некоторые из основных элементов в выше статьи, необходимо понять:
  
- ExpressRoute цепи не сопоставленные с определенным физическая инфраструктура, но являются логического подключение в одном расположении авторами, корпорация Майкрософт и авторами поставщика от вашего имени.

- Нет сопоставления 1:1 между ExpressRoute цепи и ключ s клиента.

- Каждый канал может поддерживать до 3 независимой авторами связи (Azure общедоступных авторами, Azure закрытый авторами и авторами Microsoft); Office 365 требуется Microsoft авторами.

- Каждый канал имеет основных пропускной способности, которая совместно используется все авторами связи.

- Любые открытые адреса IPv4 и общей в качестве номера, который будет использоваться для ExpressRoute цепи должен быть проверку принадлежащие или назначенные только владелец диапазон адресов.

- Виртуальные каналы ExpressRoute являются избыточными глобально и будет придерживаться стандартного протокола BGP маршрутизации. Вот почему мы рекомендуем два физических каналов в выходной к поставщику в конфигурации активный/активный.

Перейдите на [часто Задаваемые вопросы](https://azure.microsoft.com/documentation/articles/expressroute-faqs/) , Дополнительные сведения о службах поддерживается, затраты и сведения о конфигурации. Обратитесь к [статье ExpressRoute расположения](https://azure.microsoft.com/documentation/articles/expressroute-locations/) сведения в списке поставщиков предложение авторами службы поддержки Майкрософт. Мы также записанные 10 [ExpressRoute Azure для обучающие курсы по Office 365](https://channel9.msdn.com/series/aer) серии на Channel 9, чтобы объяснить понятиями более подробно.

|Консолидация малого офиса  <br/> |Доступные пропускной способности и данные об использовании следует тщательно сравнивать.  <br/> |

> [!NOTE]
> Microsoft будет предпочитают ExpressRoute через Интернет, если маршрут доступна вне зависимости от физического расположения.

## <a name="selective-routing-with-expressroute"></a>Выборочный маршрутизации с помощью ExpressRoute

Выборочный маршрутизации с помощью ExpressRoute может быть необходим для различных причин, например, тестирование, развертывание ExpressRoute к подмножеству пользователей. Существуют различные средства, которые пользователи могут применять выборочно маршрутизацию сетевого трафика Office 365 через ExpressRoute:
  
1. **Фильтрация маршрут в разделения** - разрешение маршрутов протокола BGP в Office 365 через ExpressRoute подмножество подсети или маршрутизаторы. Маршрутизация выборочно сегменте сети клиента или расположение реального офиса. Это распространенные поэтапного развертывания ExpressRoute для Office 365, настроенных на устройствах протокола BGP.

2. **Файлы PAC/URL-адреса** - со ссылкой на Office 365, предназначенные сетевого трафика для конкретных полные доменные имена для маршрутизации по указанному пути. Это выборочно могут направляет в с клиентского компьютера, идентифицируемый [PAC файла развертывания](https://aka.ms/manageo365endpoints#ID0EACAAA=2._Proxies).

3. **Фильтрация маршрутов** - [фильтры маршрутов](https://docs.microsoft.com/azure/expressroute/how-to-routefilter-portal) — это способ использования подмножества поддерживаемые службы с помощью Microsoft авторами.

4. **Сообщества протокола BGP** - фильтрации в зависимости от [протокола BGP сообщества тегов](https://aka.ms/bgpexpressroute365) позволяет клиента определить, какие приложения Office 365 должны пройти ExpressRoute и которого будет проходят через Интернет.
