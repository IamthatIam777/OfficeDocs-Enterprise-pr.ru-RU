# <a name="upgrading-from-sharepoint-2010"></a>Обновление с SharePoint 2010

На 13 центра развертывания Office 2020 г. Microsoft Office SharePoint Server 2010 достигает окончание поддержки. В этой статье приведены сведения о ресурсы, которые помогут пользователям миграция данных SharePoint Server 2010 в SharePoint Online или обновления SharePoint Server на локальную.
  
## <a name="what-is-end-of-support"></a>Что такое окончание поддержки?

При достижении конца его жизненного цикла поддержки (время, во время которого корпорация Майкрософт предоставляет новые функции, исправления ошибок, исправления безопасности и т.д.) программное обеспечение SharePoint Server 2010 и SharePoint Foundation 2010 этот процесс называется программного обеспечения «окончание поддержки», или, в некоторых случаях его «пенсионного обеспечения». После окончания поддержки (или EOS) продукта ничего не фактически завершает работу или перестает работать; Тем не менее, в конце поддержку программного обеспечения, корпорация Майкрософт не предоставляет:
  
- Технической поддержки для решения проблем, которые могут возникнуть;
    
- Исправления ошибок, проблем, обнаруженных и, которое может повлиять на надежность и удобство использования сервера;
    
- Исправления безопасности для уязвимостей, обнаруженных и, которое может сделать сервер подвержена нарушения безопасности;
    
- Обновления для часового пояса.
    
Это означает, будет дальнейшие обновления, исправления, или исправления будет был включен для продукта (включая исправления исправления) и службы поддержки Майкрософт будет полностью изменились борьбе поддержки в более ранних версиях. Как подходы окончание поддержки SharePoint Server 2010, следует использовать преимущества функциональных возможностей для удаления данных, которые больше не требуется до обновления продукта и/или перенос важных данных.
  
> [!NOTE]
> Десять лет с момента первоначального выпуска продукта обычно продолжается жизненного цикла программного обеспечения. Можно выполнить поиск для [Партнеров Майкрософт](https://go.microsoft.com/fwlink/?linkid=841249) , которые могут помочь при обновлении до следующей версии программного обеспечения или миграции в Office 365 (или оба). Убедитесь, что вам известны конца цикла поддержки на критические базовые технологии, особенно версия SQL server, используемого с SharePoint.

|||
|:-----|:-----|
||Последовательного обновления путь ***: SharePoint Server 2010 **\>** SharePoint Server 2013 **\>** SharePoint Server 2016 |
   
Если вы решаете выполните полный путь с SharePoint 2010 для SharePoint Server 2016, это потребует времени и планирования. Обновления включают стоимость (Помните, что серверы SQL Server также должны быть обновлены) обновленных оборудования, программного обеспечения и администрирования. Кроме того настроек может потребоваться обновить или даже заброшены пользователями. Не забудьте собирайте заметки на все критические настройки перед обновлением фермы SharePoint.
  
> [!NOTE]
> Существует возможность поддерживать вашего плана фермы поддержки SharePoint 2010, для установки фермы SharePoint Server 2016 на новом оборудовании (то есть отдельных ферм запустите рядом друг с другом) и затем планирование и выполнение ручного переноса содержимого (для загрузки и повторно передачи контента, для пример). Просмотр и ранжирования контента фермы SharePoint Server (определить, какие данные высокий, средний или низкий влияния для вашей компании) могут быть полезны в принятии решения. Возможно, что единственное, необходимую для совместного использования с SharePoint Online — (a) учетные записи для входа в систему и (b индекса поиска SharePoint Server — это может быть открытым, пока не откроете как используются на сайтах. Если позднее потребуется миграция все содержимое в SharePoint Online, можно переместить всех оставшихся учетных записей и данных через Интернет и выводить из эксплуатации локальную ферму и управления и администрирования фермы SharePoint будет осуществляться одним из Office 365 консоли с этого момента.
  
Обязательно ознакомьтесь с существующей типы гибридного и как настроить соединение между локальной фермы SharePoint и подписки Office 365.
  
Позволяет проверить работу гибридного веб-фермы SharePoint — путем создания [Office 365 dev/тестовой среды](https://go.microsoft.com/fwlink/?linkid=843152). После того как у вас есть пробную версию или приобрести подписки Office 365, вы будете работу по созданию семейств веб-сайтов, веб-сайтов и библиотек документов в SharePoint Online, к которому можно перенести данные (либо вручную, с помощью API-интерфейса миграции или — если нужно перенести Мои Контента сайта в OneDrive для бизнеса - в мастере гибридной).
  
> [!NOTE]
> Помните, что фермы SharePoint Server 2010 сначала необходимо обновить, локальная, SharePoint Server 2013 или SharePoint Server 2016 для использования параметра гибридного. SharePoint Foundation 2010 и SharePoint Foundation 2013 не удается создать гибридного подключения с использованием SharePoint Online.
