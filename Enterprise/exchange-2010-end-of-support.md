# <a name="exchange-2010-end-of-support-roadmap"></a>Конец стратегия поддержки Exchange 2010

На **14 января 2020**Exchange Server 2010 будет звонить окончание поддержки. Если еще не уже начали миграции от Exchange 2010 в Office 365 или Exchange 2016, пришло время начала планирования. 
  
## <a name="what-does-end-of-support-mean"></a>Окончание поддержки среднее какие работает?

Exchange Server, как и практически во всех продуктов Майкрософт, имеет жизненный цикл поддержки, во время которого предоставляются новые функции, исправления ошибок, исправления безопасности и т. д. Обычно этот жизненный цикл продолжается десять лет с момента первоначального выпуска продукта и конечных этот жизненный цикл называется окончание поддержки продукта. При достижении Exchange 2010 его окончание поддержки на 14 января 2020 г., корпорация Майкрософт больше не предоставляет:
  
- Технической поддержки для решения проблем, которые могут возникнуть;
    
- Исправления ошибок, проблем, обнаруженных и, которое может повлиять на надежность и удобство использования сервера;
    
- Исправления безопасности для уязвимостей, обнаруженных и, которое может сделать сервер подвержена нарушения безопасности;
    
- Обновления для часового пояса.
    
Установки Exchange 2010 будут выполняться после указанной даты. Тем не менее из-за изменения, перечисленные выше, мы настоятельно рекомендуем выполнить миграцию из Exchange 2010 как можно скорее.
  
Дополнительные сведения о близко к окончание поддержки Office 2010 на серверах можно [ресурсы, которые помогут вам обновление с Office 2010 серверов и клиентов](upgrade-from-office-2010-servers-and-products.md).
  
## <a name="what-are-my-options"></a>Что такое параметры?

В Exchange 2010 достигает его окончание поддержки это много времени для просмотра параметров и Подготовка плана миграции. Вы можете:
  
- Перенос в Office 365 с помощью прямой, express или гибридной миграции; 
    
- Перенос серверов Exchange 2010 до новой версии Exchange на локальных серверах.
    
Следующие разделы посвящены каждого варианта более подробно.
  
### <a name="migrate-to-office-365"></a>Миграция в Office 365

Миграция электронной почты в Office 365 — вашей наиболее и простой помогут вам удаление развертывания Exchange 2010. С помощью миграции в Office 365 можно сделать одного перехода от старой технологии современные возможности, как:
  
- Соответствие требованиям возможности, такие как политики хранения на месте и хранение для судебного разбирательства, на месте обнаружения электронных данных и многое другое;
    
- Группами Майкрософт;
    
- Power BI;
    
- Конкретный папки "Входящие";
    
- Углубимся аналитики;
    
- API-интерфейсы REST для программного доступа к электронной почты, календари, контакты и т. д.
    
Office 365 также получает новые функции и каждый раз первый и пользователи обычно можно начать использовать их немедленно. В дополнение к новые компоненты не должны тратить:
  
- Приобретение и обслуживание оборудования;
    
- Оплаты отопления и вентиляции серверов;
    
- Установка последних на исправления безопасности, продукта и часового пояса;
    
- Обслуживание хранилища и программного обеспечения для поддержки соответствия требованиям;
    
- Обновление до новой версии Exchange - вы всегда в последней версии Exchange в Office 365.

#### <a name="cutover-migration"></a>Прямая миграция

Прямая миграция — это один где, предварительно выбранную дату и время, производиться миграция почтовых ящиков, групп рассылки, контакты и т. д в Office 365; После завершения, вы будете завершение работы локальных серверов Exchange Server и начать исключительно с помощью Office 365.
  
Метод прямой миграции отлично подходит для небольших организаций, имеющих очень числа почтовых ящиков, чтобы быстро приступить к Office 365, а не требуется решить некоторые сложности других методов. Но он также несколько ограничены, поскольку оно должно быть меньше или завершенных в течение недели и пользователям необходимо повторно настройте свой профиль Outlook. При прямой миграции может обрабатывать размером до 2 000 почтовых ящиков, настоятельно рекомендуется перенести не более 150 почтовых ящиков с помощью этого метода. При попытке выполнить миграцию более 150 почтовые ящики, можно запустить из него времени для переноса всех почтовых ящиков перед вашей крайний срок и множества помочь ИТ-поддержки персонала может получить пользователей выполняет настройку Outlook.
  
Если вы собираетесь выполнив прямой миграции, вот учесть несколько моментов, учитывайте следующее:
  
- Office 365, необходимо подключиться к серверах Exchange 2010 с помощью мобильного Outlook через TCP-порт 443;
    
- Все локальные почтовые ящики будут перемещены в Office 365;
    
- Вам потребуются локального администратора, имеет права на чтение почтовые ящики пользователей;
    
- Exchange 2010 обслуживаемые домены, которые вы хотите использовать в Office 365 требуется для добавления в качестве подтвержденным доменов в службе;
    
- С момента запуска переноса и при начале этапа завершения Office 365 будет периодически выполнять синхронизацию Office 365 и локальных почтовых ящиков. Это позволяет завершения миграции не причинят электронной почты, оставленных в локальных почтовых ящиков;
    
- Пользователи будут получать новые временные пароли для своей учетной записи Office 365, который необходимо изменить при входе в к своим почтовым ящикам в первый раз;
    
- Необходимые лицензии Office 365, которая включает в себя Exchange Online для каждого почтового ящика пользователя, который миграции;
    
- Пользователям потребуется настроить новый профиль Outlook на всех своих устройств и снова загрузите своей электронной почте. Объем электронной почты, который будет загрузить Outlook могут различаться. Для получения дополнительных сведений рассмотрим [Изменение объема почты для хранения в автономном режиме](https://support.office.com/en-us/article/Change-how-much-mail-to-keep-offline-f3a1251c-6dd5-4208-aef9-7c8c9522d633?ui=en-US&amp;rs=en-US&amp;ad=US&amp;fromAR=1).
    
Чтобы узнать больше о прямой миграции, посмотрите на:
  
- [Что необходимо знать о прямой миграции электронной почты в Office 365](https://support.office.com/en-us/article/What-you-need-to-know-about-a-cutover-email-migration-to-Office-365-961978ef-f434-472d-a811-1801733869da)
    
- [Выполнение прямой миграции электронной почты в Office 365](https://support.office.com/en-us/article/Perform-a-cutover-migration-of-email-to-Office-365-9496e93c-1e59-41a8-9bb3-6e8df0cd81b4)
    
#### <a name="express-migration"></a>Express миграции

Быстрая миграция — это один при наличии нескольких сотен почтовые ящики требуется мигрировать на Office 365, можно выполнить перенос в течение нескольких недель и не требуется ни один из функции расширенные гибридной миграции как общих сведений календаря о доступности.
  
Быстрая миграция отлично подходит для организаций, которым требуется больше времени для перемещения их почтовых ящиков в Office 365, но по-прежнему планирование для выполнения миграции в течение нескольких недель. Получите некоторые преимущества дополнительных полный гибридная Миграция без многие сложности. Можно управлять сколько и какие, почтовые ящики переносятся в указанное время; Почтовые ящики Office 365 будут создаваться с именем пользователя и пароли для своих локальных учетных записей; и, в отличие от прямая миграция пользователям не нужно будет повторно создать свой профиль Outlook.
  
Если вы собираетесь выполнив поэтапной миграции, Вот несколько вещей, которые следует учитывать.
  
- Office 365, необходимо подключиться к серверах Exchange 2010 с помощью мобильного Outlook через TCP-порт 443;
    
- Вам потребуется выполнить синхронизацию одноразовый каталогов между локальными серверами Active Directory и Office 365;
    
- Пользователи смогут войти свой почтовый ящик Office 365 с помощью же имя пользователя и пароль, под которыми они использовали при их почтовых ящиков была перенесена;
    
- Необходимые лицензии Office 365, которая включает в себя Exchange Online для каждого почтового ящика пользователя, который миграции;
    
- Пользователям не нужно настроить новый профиль Outlook на большая часть свои устройства (некоторые старые Android телефоны может потребоваться новый профиль) и не нужно будет повторно загрузить своей электронной почте.
    
Чтобы узнать больше о поэтапной миграции, обратите внимание на [Использование минимальной гибридного быстро миграции почтовых ящиков Exchange в Office 365](https://support.office.com/article/Use-Minimal-Hybrid-to-quickly-migrate-Exchange-mailboxes-to-Office-365-fdecceed-0702-4af3-85be-f2a0013937ef)
  
#### <a name="full-hybrid"></a>Полный гибридного

Полный гибридной миграции, где вашей организации есть несколько сотен до десятков тысяч почтовых ящиков и необходимо переместить некоторые или все из них в Office 365. Поскольку эти миграций, как правило долгосрочные, гибридные миграции позволяют:
  
- Отображение сведений о доступности для пользователей локальных пользователей в Office 365 и наоборот;
    
- Просмотреть единый глобальный список адресов, содержащую получателей в локальной и Office 365;
    
- Просмотр полного карт получателей Outlook для всех пользователей, независимо от того, является ли они в локальной организации или в Office 365;
    
- Безопасный обмен данными электронной почты между локальными серверами Exchange и Office 365 с использованием TLS и сертификаты;
    
- Обрабатывать сообщения, отправленные между локальными серверами Exchange и Office 365 являются внутренними, позволяя:
    
  - Необходимо правильно оценить и обработанных агентом транспорта и обеспечения соответствия требованиям, предназначенные для внутренних сообщений;
    
  - Обход фильтров защиты от нежелательной почты.
    
Полный гибридные миграции, лучше всего подходит для организаций, которые должны быть в курсе в гибридной конфигурации для многих месяцев или нескольких. Вы получите компоненты, перечисленными ранее в этом разделе, а также синхронизации службы каталогов, лучше функции интегрированной контроля соответствия требованиям и возможность перемещать почтовые ящики на и с Office 365 с помощью перемещения почтовых ящиков online. Office 365 становится расширение локальной организации.
  
Если вы собираетесь выполните полный гибридной миграции, Вот несколько вещей, которые следует учитывать.
  
- Полный гибридные миграции не подходят для всех типов организаций. Из-за сложности полный гибридные миграции организаций с меньше, чем несколько сотен почтовые ящики не обычно отображается преимуществ, которые обоснование усилий и расходы, необходимые для настройки одного. Если это может показаться вашей организации, настоятельно рекомендуется учитывать Cutover или Поэтапная миграция вместо;
    
- Office 365, необходимо подключиться к серверу Exchange 2010 с помощью мобильного Outlook через TCP-порт 443;
    
- Вам потребуется Настройка синхронизации службы каталогов с помощью Active Directory, подключения Azure (AADConnect) между локальными серверами Active Directory и Office 365;
    
- Пользователи смогут войти свой почтовый ящик Office 365 с помощью же имя пользователя и пароль, которые они используют при входе в локальной сети (требуется подключение Azure Active Directory с помощью синхронизации паролей и/или служб федерации Active Directory);
    
- Необходимые лицензии Office 365, которая включает в себя Exchange Online для каждого почтового ящика пользователя, который миграции;
    
- Пользователям не нужно настроить новый профиль Outlook на большая часть свои устройства (некоторые старые Android телефоны может потребоваться новый профиль) и не нужно будет повторно загрузить своей электронной почте.
    
Если полный гибридной миграции звуки подходящую для вас, рассмотрим следующие ресурсы, которые помогут в работе с миграции:

### <a name="migrate-to-a-newer-version-of-exchange-server"></a>Перенос в новой версии Exchange Server

Хотя мы настоятельно считаем, что можно достичь оптимального значение и пользователя, миграции в Office 365, также понять, что в некоторых организациях требуется хранить свои электронной почты для локальной. Это может быть из-за нормативным требованиям, чтобы гарантировать данные не хранятся в центре обработки данных, расположенного в другой стране и т. д. При выборе для хранения электронной почты на локальную можно выполнить migrate среды Exchange 2010 в Exchange 2013 или Exchange 2016.
  
Мы рекомендуем миграции на Exchange 2016, если не удается выполнить миграцию с Office 365. Exchange 2016 включает все функции и усовершенствования, включенные в предыдущие версии Exchange, и наиболее близко соответствует удобство работы с Office 365 (несмотря на то, что некоторые функции, доступные только в Office 365). Извлечение только некоторые задачи, которые вы отсутствует на:
  
|**Версии Exchange**|**Возможности**|
|:-----|:-----|
|Exchange 2013  <br/> | Упрощенная архитектура, уменьшение числа ролей сервера в три (транспорта пограничного сервера почтовых ящиков, клиентского доступа)  <br/>  Данные политик предотвращения потери (DLP), которые помогают хранения конфиденциальной информации от утечки  <br/>  Значительно Outlook Web App улучшают  <br/> |
|Exchange 2016  <br/> | *Функции из Exchange 2013 и...*  <br/>  Дополнительные роли сервера упрощенный для только что почтовых ящиков и пограничных транспортных  <br/>  Улучшенная защиты от потери данных и интеграции с SharePoint  <br/>  Устойчивость улучшенные базы данных  <br/>  Совместная работа над документами через Интернет  <br/> |

Мы рекомендуем, что изначально предполагается, что производиться миграция к Exchange 2016. Затем используйте следующие сведения для подтверждения вашей допущения или исключить Exchange 2016. Если не удается переходе на Exchange 2016 для той или иной причине, же обработки с Exchange 2013 и так далее.
  
|**Рекомендации**|**Дополнительная информация**|
|:-----|:-----|
|Конец цикла поддержки  <br/> | Как и Exchange 2010 каждой версией Exchange имеет свой собственный поддержки Дата завершения:  <br/> **Exchange 2013** - 2023 апреля  <br/> **Exchange 2016** — 2025 октября  <br/>  Ранее в конце поддержка даты, sooner вам потребуется выполнение миграции с другой. Апрель 2023 гораздо более подробно, чем вы думаете!<br/> |
|Путь миграции Exchange 2013 или 2016  <br/> |Ниже приведены общие этапы для миграции в Exchange 2013.  <br/> Установка Exchange 2013 или 2016 в существующих служб Move организации Exchange 2010 и других инфраструктуру для Exchange 2013 или 2016 перемещения почтовых ящиков и общих папок в Exchange 2013 или Decommission 2016 оставшихся серверах Exchange 2010 |
|Версия сосуществования  <br/> |При миграции в Exchange 2013 или Exchange 2016, либо версию можно установить в существующей организации Exchange 2010. Это позволяет установить один или несколько серверов Exchange 2013 или Exchange 2016 и выполнять миграцию.  <br/> |
|Серверное оборудование  <br/> | Требования к оборудованию сервера изменились из Exchange 2010. Вам потребуется убедитесь, что совместим оборудования, которое вы собираетесь использовать. Можно найти дополнительные сведения о требования к оборудованию для каждой версии:<br/> [Системные требования Exchange 2016](https://technet.microsoft.com/en-us/library/aa996719%28v=exchg.160%29.aspx) <br/> [Системные требования Exchange 2013](https://technet.microsoft.com/en-us/library/aa996719%28v=exchg.150%29.aspx) <br/>  Можно найти, что с значительные улучшения в производительности Exchange и вычислительных и емкость хранилища в новых серверов, скорее всего вам будет меньшего числа серверов для поддержки такое же число почтовых ящиков.  <br/> |
|Версия операционной системы  <br/> | Минимальная поддерживаемая операционная система версии для каждой версии являются:  <br/> **Exchange 2016** Windows Server 2012 г.  <br/> **Exchange 2013** Windows Server 2008 R2 с пакетом обновления 1  <br/>  Дополнительные сведения о поддержке операционной системы можно найти в [Таблица поддержки Exchange](https://technet.microsoft.com/en-us/library/ff728623%28v=exchg.150%29.aspx).  <br/> |
|Функциональный уровень леса Active Directory  <br/> | Приведены минимальные поддерживаемые Active Directory работы леса для каждой из версий.  <br/> **Exchange 2016** Windows Server 2008 R2 с пакетом обновления 1  <br/> **Exchange 2013** Windows Server 2003  <br/>  Дополнительные сведения о поддержке функциональный уровень леса можно найти в [Таблица поддержки Exchange](https://technet.microsoft.com/en-us/library/ff728623%28v=exchg.150%29.aspx).  <br/> |
|Версии клиента Office  <br/> | Приведены минимальные поддерживаемые версии клиента Office для каждой из версий.  <br/> **Exchange 2016** Office 2010 (с последними обновлениями)  <br/> **Exchange 2013** Office 2007 с пакетом обновления 3  <br/>  Дополнительные сведения о поддержке клиентов Office можно найти в [Таблица поддержки Exchange](https://technet.microsoft.com/en-us/library/ff728623%28v=exchg.150%29.aspx).  <br/> |
