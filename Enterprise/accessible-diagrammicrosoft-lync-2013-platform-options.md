# Текстовая версия схемы, в которой представлены варианты платформы Microsoft Lync 2013

Эта статья  удобная текстовая версия схемы, показывающей варианты платформы Microsoft Lync 2013, которая доступна в статье [Технические схемы](http://go.microsoft.com/fwlink/?LinkID=519139).
  
На этом плакате представлены сведения о вариантах развертывания Lync Online (Office 365) и Lync Server, которые должны знать архитекторы и лица, принимающие бизнес-решения.
  
- Сравнение четырех различных вариантов развертывания: Lync Online (Office 365), гибридная среда Lync Online и Lync Server, Lync Server и Lync Server с размещением у поставщика. 
    
- Два примера сценариев развертывания Lync 2013.
    
## Сравнение четырех различных вариантов развертывания платформы Lync 2013

В сравнении по каждому варианту развертывания представлена следующая информация:
  
- обзор различных компонентов развертывания;
    
- преимущества каждого из вариантов развертывания;
    
- требования лицензирования;
    
- необходимые архитектурные задачи;
    
- обязанности ИТ-специалистов по внедрению каждого варианта развертывания.
    
### Общие сведения

#### Lync Online (Office 365)

Повысьте эффективность работы и сократите затраты с помощью мультитенантных планов Office 365.
  
На сопроводительной схеме показана служба Lync Online с клиентом Azure Active Directory, который синхронизирует имена и пароли учетных записей между локальной средой доменных служб Active Directory и клиентом Azure Active Directory.
  
Описание компонентов и функций:
  
- Программное обеспечение как услуга (SaaS).
    
- Большой набор всегда обновленных компонентов.
    
- В том числе и клиент Azure Active Directory для учетных записей интернет-служб, который можно использовать с другими приложениями. 
    
- Благодаря функции интеграции каталогов можно синхронизировать имена и пароли учетных записей между локальными доменными службами Active Directory и клиентом Azure Active Directory.
    
- Если необходима функция единого входа, потребуется реализовать службы федерации Active Directory. 
    
- Данные, которыми клиент обменивается через Интернет, шифруются. При их передаче используется аутентификация.
    
- Возможность подключения устаревшего телефонного оборудования (телефонной сети общего пользования, ТСОП) с помощью сторонних поставщиков.
    
#### Гибридная среда Lync Online и Lync Server (с разделенным доменом)

Вы можете использовать преимущества Office 365 и при локальном развертывании Lync 2013.
  
На сопроводительной схеме показаны Office 365 с Lync Online, при этом часть пользователей размещена в локальной среде, а другая часть  в Интернете. Кроме того, на схеме имеется пограничный сервер, развернутый в локальной среде.
  
Описание компонентов и функций:
  
- Часть пользователей размещена в локальной среде, а другая часть  в Интернете, но все они используют один домен SIP, например contoso.com.
    
- Использование существующей инфраструктуры Lync Server 2013, в том числе подключений к ТСОП.
    
- Простое добавление новых пользователей Lync Online, которым не требуется доступ к ТСОП.
    
- Миграция из локальной среды Lync в Lync Online согласно заданному вами расписанию.
    
- Интеграция с другими приложениями Office 365, в том числе с Exchange Online и SharePoint Online.
    
#### Lync Server

При таком варианте развертывания все ресурсы находятся под вашим контролем. На сопроводительной схеме показана инфраструктура Lync Server с локальной средой доменных служб Active Directory, в которой пользователи размещены локально.
  
Описание компонентов и функций:
  
- Планирование загрузки и определение конфигурации оборудования.
    
- Приобретение и настройка серверов.
    
- Развертывание.
    
- Горизонтальное масштабирование, применение исправлений и эксплуатация.
    
- Резервное копирование данных.
    
- Обслуживание средств отработки отказа и аварийного восстановления.
    
- Подключение инфраструктуры Lync Server 2013 к ТСОП.
    
- Интеграция с существующим телефонным оборудованием, например с частными УАТС.
    
#### Lync Server с размещением у поставщика

При таком варианте развертывания все ресурсы находятся под контролем поставщика. На сопроводительной схеме показана сеть поставщика, включающая серверы и оборудование, подключенная к локальной среде.
  
Описание компонентов и функций:
  
- Планирование загрузки и определение конфигурации оборудования.
    
- Приобретение и настройка серверов.
    
- Развертывание.
    
- Горизонтальное масштабирование, применение исправлений и эксплуатация.
    
- Резервное копирование данных.
    
- Обслуживание средств отработки отказа и аварийного восстановления.
    
- Интеграция с существующим телефонным оборудованием, например с частными УАТС.
    
- Кроме того, поставщик может:
    
  - Установить свои серверы и оборудование в собственной сети с подключением к вашей локальной среде (сплошная линия).
    
  - Установить свои серверы в вашей локальной среде (пунктирная линия).
    
### Преимущества каждого из вариантов развертывания

#### Lync Online (Office 365)

- Отсутствие операционных затрат на локальные серверы или серверное программное обеспечение.
    
- Возможности для общения, доступные благодаря облачной службе Lync Server 2013.
    
- Lync предоставляет такие возможности: получение сведений о присутствии, обмен мгновенными сообщениями, использование аудио- и видеосвязи, проведение собраний по сети и веб-конференций.
    
- Данный вариант используется географически разрозненными организациями или организациями с большим количеством мобильных сотрудников.
    
#### Гибридная среда Lync Online и Lync Server (с разделенным доменом)

- Использование Lync Online для связи с удаленными пользователями и деловыми партнерами.
    
- Упрощение миграции из локальной среды Lync в Lync Online.
    
- Связь с филиалами без использования соответствующих устройств.
    
- Простота внедрения Lync для новых подразделений организации.
    
#### Lync Server

- Решения для частного облака.
    
- Решения с широкими возможностями настройки.
    
- Lync Online не поддерживает устаревшие решения со сторонними компонентами, зависящими от оборудования и программного обеспечения.
    
- Ограничения конфиденциальности, мешающие синхронизации учетных записей доменных служб Active Directory с Office 365.
    
- Организации, которым необходим контроль над всей платформой и решением.
    
- Замена УАТС на Корпоративную голосовую связь Lync.
    
#### Lync Server с размещением у поставщика

- Организации, которым необходимо использовать возможности Lync Server, но так, чтобы развертыванием и обслуживанием занималась другая компания.
    
- Решения с привлечением поставщика.
    
- Решения с широкими возможностями настройки.
    
- Lync Online не поддерживает устаревшие решения со сторонними компонентами, зависящими от оборудования и программного обеспечения.
    
- Замена УАТС на Корпоративную голосовую связь Lync.
    
### Требования лицензирования

#### Lync Online (Office 365)

Модель с использованием подписки.
  
#### Гибридная среда Lync Online и Lync Server (с разделенным доменом)

- Для Office 365: модель с использованием подписки. Дополнительные лицензии не требуются. 
    
- Для локальной среды: можно применять все локальные лицензии. 
    
#### Lync Server

- Серверная операционная система
    
- SQL Server
    
- Серверная лицензия для Lync 2013
    
- Клиентская лицензия для Lync 2013
    
#### Lync Server с размещением у поставщика

Затраты зависят от соглашения с поставщиком решения Lync.
  
### Архитектурные задачи

В этом разделе перечислены архитектурные задачи для каждого варианта развертывания.
  
#### Lync Online (Office 365)

- Планирование и проектирование синхронизации каталогов.
    
- Обеспечение доступности и достаточной пропускной способности сети при использовании брандмауэров, прокси-серверов, шлюзов и каналов глобальной сети.
    
- Приобретение сторонних SSL-сертификатов, позволяющих обеспечить защиту корпоративного уровня для служб Office 365.
    
- Принятие решения о том, необходимо ли подключаться к Office 365 по протоколу IP версии 6 (IPv6).
    
#### Гибридная среда Lync Online и Lync Server (с разделенным доменом)

Кроме решения задач для среды Office 365 и локальной среды, нужно:
  
- Определить необходимую степень интеграции компонентов для локальных и веб-версий Exchange Server и SharePoint.
    
- При необходимости следует определить, какое устройство прокси-сервера будет использоваться для запросов из Office 365.
    
#### Lync Server

Проектирование среды Lync в существующей локальной среде.
  
- Топология Lync для центрального офиса и филиалов.
    
- Серверное оборудование, в том числе для виртуализации.
    
- Интеграция с доменными службами Active Directory и DNS.
    
- Балансировка нагрузки для пулов серверов Lync.
    
- Отработка отказа и аварийное восстановление.
    
#### Lync Server с размещением у поставщика

- В случае установки в облачной среде необходимо определить способ подключения к сети поставщика услуг.
    
- В случае установки в локальной среде необходимо определить способ размещения серверов Lync поставщика в вашей сети.
    
- В случае использования обоих типов установки необходимо определить способ интеграции с доменными службами Active Directory и оборудованием УАТС.
    
### Обязанности ИТ-специалистов

В этом разделе перечислены обязанности ИТ-специалистов по внедрению каждого варианта развертывания.
  
#### Lync Online (Office 365)

Развертывание службы Lync Online (Office 365) и управление ею.
  
- Внедрение плана синхронизации каталогов.
    
- Планирование внутренних и внешних записей DNS и маршрутизации.
    
- Настройка прокси-сервера или брандмауэра согласно требованиям Office 365 к IP-адресу и URL-адресам.
    
- Администрирование учетных записей пользователей и параметров Lync Online.
    
#### Гибридная среда Lync Online и Lync Server (с разделенным доменом)

Кроме решения задач для среды Office 365 и локальной среды, нужно:
  
- Настроить устройства прокси-сервера (при необходимости).
    
- Настроить интеграцию компонентов с локальными и веб-версиями Exchange Server и SharePoint.
    
#### Lync Server

Развертывание Lync Server в локальной среде и управление им.
  
- Подготовка серверов к работе.
    
- Развертывание топологии Lync.
    
- Обновление серверов Lync.
    
- Добавление серверов топологии в ферму или их удаление в зависимости от загруженности ресурсов.
    
- Реализация среды для отработки отказа и аварийного восстановления.
    
#### Lync Server с размещением у поставщика

Задачи, которые нужно выполнить вместе с поставщиком:
  
- Интеграция Lync Server в вашу сеть.
    
- Интеграция Lync Server с другими продуктами Майкрософт или с собственными решениями.
    
- Отслеживание выполнения соглашений об уровне обслуживания поставщиком.
    
## Два примера сценариев развертывания Lync 2013.

### Компоненты средства синхронизации каталогов в Microsoft Azure

Развертывание компонентов для синхронизации каталогов Office 365 в Azure происходит быстрее, так как виртуальные машины можно развертывать по запросу.
  
На сопроводительной схеме показана служба Lync Online с клиентом Azure Active Directory, которая синхронизирует имена и пароли учетных записей между локальной средой Active Directory и клиентом Azure Active Directory.
  
 **Только сервер синхронизации каталогов**. Чтобы не развертывать в локальной среде 64-разрядную версию сервера синхронизации каталогов, можно через Интернет подготовить к работе виртуальную машину в Azure.
  
 **Средство синхронизации каталогов и службы федерации Active Directory**. Этот вариант позволяет поддерживать федеративные удостоверения Office 365, не добавляя оборудование в локальную инфраструктуру. Кроме того, при использовании этого варианта обеспечивается устойчивость системы в случае недоступности локальной среды Active Directory. Ниже перечислены возможности, доступные при использовании этого варианта.
  
- Компоненты интеграции каталогов работают как виртуальные машины Azure.
    
- Службы федерации Active Directory публикуются в Интернете через прокси-серверы этих служб, работающие как виртуальные машины Azure.
    
- Для пользователей, подключающихся из любого расположения, трафик проверки подлинности клиентов обрабатывается серверами службы федерации Active Directory и прокси-серверами, развернутыми в качестве виртуальных машин Azure.
    
### Интеграция Lync с Exchange и SharePoint в Office 365

#### Lync Server с Exchange Online и SharePoint Online

На сопроводительной схеме вы можете видеть Office 365 с Exchange Online и SharePoint Online, подключенными к локальной ферме Lync Server 2013.
  
При таком варианте развертывания можно:
  
- Использовать все возможности Lync Server 2013.
    
- Использовать существующее локальное телефонное оборудование, например УАТС.
    
- Использовать Exchange Online для электронной почты и уменьшения нагрузки на локальные почтовые серверы и хранилища.
    
- Использовать SharePoint Online для совместной работы и уменьшения нагрузки по обслуживанию локальных серверов SharePoint.
    
- Использовать интегрированные функции Lync, Exchange и SharePoint, включая единую систему обмена сообщениями, в Office 365.
    
#### Exchange Server с Lync Online

На сопроводительной схеме можно увидеть Office 365 с Lync Online, подключенные к локальной ферме Exchange Server.
  
При таком варианте развертывания можно:
  
- Использовать существующую инфраструктуру Exchange.
    
- Использовать Lync Online для получения сведений о присутствии, обмена мгновенными сообщениями и проведения конференций.
