# <a name="sharepoint-2007-migration-options-to-consider"></a>Варианты перехода SharePoint 2007, которые следует учитывать

Окончание поддержки достигнуто Microsoft SharePoint 2007 и SharePoint Server 2007. Это время для обновления! В этой статье представлены сведения о параметрах миграции.
  
## <a name="common-upgrade-strategies-for-sharepoint"></a>Общие стратегии обновления для SharePoint

Существует несколько способов обновления в среде SharePoint Server. Если у вас есть ферма Microsoft Office SharePoint Server 2007, Вот несколько примеров из методов обновления:
  
- Присоединение баз данных
    
- Обновление рядом друг с другом
    
- Обновление на месте
    
- Гибридного обновления (на месте с отсоединенной базой данных и отдельные с подключением баз данных)
    
- Гибридные устройства SharePoint (подключиться к Интернету для локального развертывания SharePoint)
    
- Вручную переместите данных между семейств веб-сайтов или библиотек
    
- Она мастер обновления до Office 365 ([SharePoint Online помощник по развертыванию](https://aka.ms/spoguidance))
    
- Программный Интерфейс переноса в SharePoint Online (SPO) в Office 365
    
Что лучше всего подходит для вас?
  
Знание что фермы и используется для при тактические силы доходит до обновления. Способ люди использовать в ферме SharePoint поможет вам выбрать один из вашего параметров.
  
> [!TIP]
> Microsoft Office SharePoint Server 2007 также имеет постепенное обновление, не представленных здесь. Список конкретных шаг обновления статьи см [конец SharePoint Server 2007 поддерживает схема](sharepoint-2007-end-of-support.md). 
  
Не забудьте проверить [Жизненный цикл продукта](https://support.microsoft.com/en-us/lifecycle/search) и требования к системе для любой версии SharePoint вы обновляете. Это позволяет принять во внимание при следующего обновления будет требоваться (например, Пауза в старых продуктов, как SharePoint Server 2010 на планирование нескольких обновлений, убедитесь, что вы знаете его поддержки Дата завершения) и, чтобы убедиться в наличии аппаратного обеспечения для плана. 
  
Если вы планируете переход некоторые или все сайты SharePoint с Office 365 в облаке, это время для закладки ссылки на [Описания служб Office 365](https://technet.microsoft.com/en-us/library/office-365-service-descriptions.aspx). Вам потребуются описания служб, чтобы узнать о возможности SharePoint Online и как они могут отличаться от локального сервера SharePoint. Обновление ферм работы Microsoft Office SharePoint Server 2007. Если установки сайты, которые будут недействительны, исправьте их перед обновлением.

### <a name="first-know-your-farm"></a>Во-первых знать фермы

При выполнении обновления, принятия должен быть основан на что означает фермы для вашей организации. Какие потребности он обеспечивает? Что такое его роли? Каждая ферма в вашей компании могут иметь разные роли. Некоторые из фермы SharePoint могут быть *критические* , некоторые могут быть архивами файлов — там для безопасного хранения. Или, если фермы заполняется много ролей за один раз, затем может потребоваться узнать, какие семейств веб-сайтов, веб-сайтов или даже библиотеки документов выполните, копирование настроек, и как они важны. Анализ данных на этом уровне может показаться много времени, но он сохраняет время и силы к собственным доменом перед обновлением или перенос, его. Узнав подвижных частей и наиболее важные бит, это также означает опередило сведения и можно оставить. Эти знания только выиграют Переход вперед. 
  
Таким образом что пользователи говорят наиболее важных о фермы SharePoint?
  
- Встроенных функций SharePoint
    
- В собрании большие объемы данных (например, архив файлов)
    
- Доступность
    
- Критически важных приложений, веб-частей или документов в ферме (поддержка критически важных ферма)

### <a name="a-simple-sample-plan"></a>Простой пример плана

Наличие может потребоваться всей согласования с руководством и другие "Администраторы" на указанный путь обновления SharePoint. Администраторов SharePoint Server часто поддерживает работу с Microsoft SQL Server "Администраторы", работа с группами безопасности и сети и многое другое. Там, где существует множество заинтересованных лиц, может потребоваться построения соглашением о или корректировка плана по обновлению и миграции. Например при миграции данных, чтобы частью компания использует SharePoint Online в Office 365 существует может понадобиться настройка производительности или тестирования внутри сети. Заблаговременное следует проинформировать затронутых группами.
  
В мой простой пример я Показать предложения администратору SharePoint, а затем список плана, согласованы всех заинтересованных лиц. Во избежание недоразумений документов соглашения и решения.
  
Планирование начинается после глубокий анализ фермы и пытается определить роль фермы, сложностями и другие важные сведения, приведет к их некоторые варианты обновления. После этого обновления предложения выполняется администратором SharePoint, а заинтересованные стороны согласны плана действий.
  
Список «наиболее важные» маркера:
  
- Доступность, возможности, встроенные в SharePoint и соответствие требованиям стандартов.
    
- Большая часть данных — на трех семейств веб-сайтов, с одной область для собраний используется группой разработчиков особенно важно, а также в интенсивное использование в различных часовых поясах по всему миру.
    
- Существует семнадцать другие веб-сайты, которые широко распространены.
    
- Две библиотеки документов (рабочая область для собраний и документы на корневого семейства сайтов), наибольшее (более чем 8000 документы каждого). У нас есть много архивных документов и списков с вложениями электронной таблицы.

- Некоторые из этих данных должно находиться в локальной, из-за правил безопасности информации.
