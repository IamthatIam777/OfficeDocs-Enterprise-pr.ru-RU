# <a name="office-365-video-networking-frequently-asked-questions"></a>Office 365 видео сети часто задаваемые вопросы

Репозитория Office 365 видео и потоковая передача служб упрощают хранения и потоковой передачи видео в вашей организации. Существует множество веских [сведения об Office 365 видео](https://support.office.com/article/Find-help-about-Office-365-Video-b435f99a-f47e-4ebd-a946-f5c965844f50); предоставляет этой сети часто задаваемые вопросы о ответить на часто задаваемые вопросы по пропускной способности, планирование, шифрование и как служба использует [сети доставки содержимого (CDN)](https://support.office.com/article/Content-delivery-networks-0140f704-6614-49bb-aa6c-89b75dcd7f1f).
  
Если вы не уже имеют хорошее представление того, что происходит при загрузке видео или воспроизведения, посмотрите это видео мы объединить, [что происходит с видеофайла при загрузке для Office 365 видео](https://www.youtube.com/watch?v=HXSZ0jYBKlM).
  
## <a name="what-are-the-office-365-video-bandwidth-requirements"></a>Каковы требования к пропускной способности видео Office 365?

Существует множество [Поддерживаемые форматы видео](https://support.office.com/article/dd1af01c-fd8e-4640-b17b-93ee02b9b817) , который можно передать на Office 365. Каждый файл видео затем кодируются в стандартный формат с несколько различных показателей качества видео для воспроизведения. Office 365 видео использует адаптивный скорость потоковой передачи для выбора оптимальное качество воспроизведения видео на основе имеющаяся пропускная способность сети и размер видеопроигрывателя. Для этого проигрыватель изначально запрашивает низкое качество воспроизведения. Служба затем начинает отправлять видео сегменты 2 секунды для видеопроигрывателя. Проигрыватель может отправить запрос на основании насколько быстро доставляется каждого сегмента качества воспроизведения выше или ниже.
  
Адаптивный скорость потоковая передача делает все это в фоновом режиме во время воспроизведения видео с наименьшие нарушения нормального функционирования или буферизация. При воспроизведении видео видеопроигрыватель позволяет просмотра вручную переопределить качество автоматического воспроизведения для выбора определенных воспроизведение видео качества.

## <a name="how-do-cdns-help-video-playback"></a>Как CDN помочь воспроизведение видео

Если несколько пользователей из той же организации, в том же месте потока же video(s), CDN будет хранить копию этих видео в папке ближе для этого региона. С видео сохраненными, или кэшированными ближайших расположении каждый пользователь отправляет видео из расположения, наиболее близкого к их вместо расположения дальнейшей нет на месте. Видео по Office 365 использует службы Azure мультимедиа для управления, что помещена в CDN Azure и в течение какого времени. Служб Azure мультимедиа можно использовать любой из [Azure CDN расположения](https://azure.microsoft.com/documentation/articles/cdn-pop-locations/) для кэширования видео фрагментов манифестов и для нескольких дней. Если пользователи в вашей организации по-прежнему видеозаписи, посвященные кэшированные они останутся в кэше. Если не один доступа к видео на несколько дней, видео будет со временем размещения удаляться из кэша. Следующий раз, кто-то попытается смотреть видео еще раз кэшируется ближайшего CDN расположении.
  
Кто пытается видео во время содержимое кэшируется на рядом CDN преимущества от видео, расположенном ближе сейчас и в большинстве случаев меньше прыжков. Это позволяет повысить скорость воспроизведения видео; Тем не менее он не изменяет требования к сети для воспроизведения видео.
  
> [!NOTE]
> Существуют некоторые обстоятельства, такие как наш ограничений емкости было достигнуто, где видео могут быть удалены до трех дней.

## <a name="can-i-cache-the-videos-locally-for-faster-playback"></a>Можно кэшировать видео локально для более быстрого воспроизведения?

Да. Office 365 не предотвратить использование локальной сети CDN или прокси-сервер кэширования для переноса видео или другие содержимого Office 365 в локальной сети для более быстрого доступа. Существует несколько способов для внедрения решения локального кэширования в сети, метод наиболее распространенных будет использовать прокси-сервер решение, которое кэширует содержимое локально. После прокси-сервера или частной сети CDN кэширования видео фрагментов и манифесты последующие запросы для файлов, направляться через прокси-сервер или частной сети CDN, которые извлекаются из локального кэша и не, которые извлекаются из веб-сайта. Рассмотрите возможность пропускной способности, емкости и параллелизм воспроизведение видео во время планирования решения следующим образом.

## <a name="how-videos-are-encrypted-and-secured"></a>Как видеоролики зашифрован и защитой?

Office 365 видео знала, насколько важно сохранить данные конфиденциальной. [Office 365 Trust Center](https://products.office.com/business/office-365-trust-center-cloud-computing-security) описаны значительные усилия для конфиденциальность и безопасность содержимого. При воспроизведении видеоклипов важна скорость для удобства работы; Тем не менее мы не поставить под угрозу безопасности и конфиденциальности, но более скорость. Вот как мы учета скорости, безопасности и конфиденциальности.
  
Когда вы или кто-то в вашей организации отправляет новое видео, видео — выполняется зашифрован с помощью шифрования AES-128 и сохраняются в служб Azure мультимедиа. Это означает, что видеоролики зашифрованные во время передачи и статических.
  
Когда кто-то в вашей организации пытается Посмотрите видеоролик по новым, они выполните следующие действия:
  
1. Выясните, SharePoint Online разрешения на просмотр видео.

2. SharePoint Online использует файл разрешения для определения, если пользователь может просмотреть видеоролик.

3. Если они разрешены, SharePoint Online получает маркер из Azure для предоставления к видеопроигрывателя.

4. Видеопроигрыватель затем использует маркер для запроса ключа расшифровки с Azure.

5. С помощью ключа расшифровки в наличии видеопроигрыватель возможность передачи видео.
