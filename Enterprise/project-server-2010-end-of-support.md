# <a name="project-server-2010-end-of-support-roadmap"></a>Project Server 2010 конец поддержки стратегия

Заканчивается поддержка для Office 2010 серверов и приложений в 2010 и определите планов для миграции. Если в настоящее время используется Project Server 2010, обратите внимание, что он и этих связанных с ними продуктов будет следующие конец цикла поддержки:
  
|**Продукт**|**Поддержка Дата окончания**|
|:-----|:-----|
|Project Server 2010  <br/> |13 октября 2020 г.  <br/> |
|Project Portfolio Server 2010  <br/> |13 октября 2020 г.  <br/> |
|Project стандартный 2010  <br/> |13 октября 2020 г.  <br/> |
|Project профессиональный 2010  <br/> |13 октября 2020 г.  <br/> |
   
Дополнительные сведения об Office 2010 на серверах достигает пенсионного обеспечения можно [обновление с Office 2010 на серверах и клиентских продуктов](https://docs.microsoft.com/office365/enterprise/plan-upgrade-previous-versions-office).
  
## <a name="what-does-end-of-support-mean"></a>Окончание поддержки среднее какие работает?

Project Server, как практически во всех продуктов Майкрософт, имеет жизненный цикл поддержки, во время которого предоставляются новые функции, исправления ошибок, исправления безопасности и т. д. Обычно этот жизненный цикл продолжается десять лет с момента первоначального выпуска продукта и конечных этот жизненный цикл называется окончание поддержки продукта. Поскольку Project Server 2010 связаться с вами его окончание поддержки на 10 октября 2020 г., корпорация Майкрософт предоставляет больше не будет:
  
- Техническая поддержка проблем, которые могут возникнуть.
    
- Наличие проблем, обнаруженных и, которое может повлиять на надежность и удобство использования сервера исправления ошибок.
    
- Исправления безопасности для уязвимостей, обнаруженных и то может привести к серверу уязвимы к безопасности.
    
- Обновления для часового пояса.
    
Установки Project Server 2010 будут выполняться после указанной даты. Тем не менее из-за изменения, перечисленные выше, мы настоятельно рекомендуем выполнить миграцию из Project Server 2010 как можно скорее.
  
## <a name="what-are-my-options"></a>Что такое параметры?

Если вы используете Project Server 2010, необходимо изучить параметры миграции, которые являются:
  
- Перенос в Project Online
    
- Миграция на новую версию локального сервера Project Server (желательно 2019 Project Server).
    
|**Почему предпочитают перенос в Project Online**|**Почему предпочитают перенос в Project Server 2019**|
|:-----|:-----|
| У меня есть пользователей мобильных устройств.  <br/>  Расходы на перенос являются большой проблемой (оборудование, программное обеспечение, часов и меры по реализации, и т.д.).  <br/>  После миграции расходы на обслуживание моей среды являются большой проблемой (например, для автоматического обновления, гарантируется время работы, и т.д.).  <br/> | Бизнес-правил ограничить мне операционной бизнес в облаке.  <br/>  Требуется элемент управления обновлений моей среды.  <br/> |
   
> [!NOTE]
> Дополнительные сведения о параметрах перемещения с серверов Office 2010 можно [ресурсы, которые помогут вам обновление с Office 2010 серверов и клиентов](https://docs.microsoft.com/office365/enterprise/upgrade-from-office-2010-servers-and-products). Обратите внимание на то, что Project Server не поддерживает гибридной конфигурации, начиная с Project Server и Project Online не могут совместно использовать тот же самый пул ресурсов. 
  
## <a name="important-considerations-you-need-to-make-when-planning-to-migrate-from-project-server-2010"></a>Важные сведения, которые необходимо принять при планировании для переноса из Project Server 2010

Вам необходимо при планировании для переноса из Project Server 2010, учтите следующее:
  
- **Получите помощь от партнера корпорации Майкрософт** — обновление с Project Server 2010 может оказаться сложной задачей и требуется много подготовки и планирования. Его может быть особенно сложной, если не было необходимо установить и настроить Project Server 2010 изначально. К счастью существует партнеров корпорации Майкрософт, можно включить для пользователей, которые этого на жизнь ли планируется перенос Project Server 2019 или Project Online. Можно выполнить поиск партнера корпорации Майкрософт при миграции на [Центр партнеров Майкрософт](https://go.microsoft.com/fwlink/p/?linkid=841249). Поиск по ключевым словам на термин *уровня Gold проекта и управления портфелем* можно извлечь список всех Microsoft партнеров с опытом в проекте. 
    
- **Планирование настройки** - помните о том, что многие из настроек, у вас есть работа в вашей среде Project Server 2010 может не работать при миграции для Project Server 2019 или Project Online. Существует большая различия в архитектуре Project Server версии, а также обязательные операционных систем, серверов баз данных и клиентского веб-браузеров, поддерживаемых для работы с новой версией. Составить план на месте на инструкции по тестированию или перестроение настройки при необходимости в новой среде. Планирование обновления также будут хорошая возможность для проверки, если определенные настройки действительно требуется при переходе. [Создание плана текущих настроек во время обновления до SharePoint 2013](https://go.microsoft.com/fwlink/p/?linkid=842061) имеет некоторые хорошие Общие сведения о оценки и планирования текущих настроек при обновлении. 
    
- **Время и Терпение** - обновление планирование, выполнение и тестирование займет много времени и усилий, особенно при обновлении до Project Server 2019. Например при переносе из Project Server 2010 для Project Server 2019, будет сначала нужно переноса из Project Server 2010 до Project Server 2013 и проверьте данные, а затем выполните одинаковые значения при переходе на каждой последующей версии (Project Сервер 2016 и затем Project Server 2019). Можно проверить с помощью партнера корпорации Майкрософт для сравнения предполагаемых затрат с их оценки как время, необходимое для их действий и какова стоимость.

## <a name="migrate-to-a-newer-on-premises-version-of-project-server"></a>Миграция на новую версию локального сервера Project Server

Хотя мы настоятельно считаем, что можно достичь оптимального значение и пользователя, перенос в Project Online, также понять, что в некоторых организациях требуется хранить данные проекта в локальной среде. Если нужно сохранить проект данных локального может перенести среды Project Server 2010, Project Server 2013, Project Server 2016 или Project Server 2019.
  
Мы рекомендуем перенос в Project Server 2019, если не удается выполнить миграцию в Project Online. Project Server 2019 включает большинство ключа, функции и усовершенствования, включенные в предыдущие версии Project Server и наиболее близко соответствует удобство работы с Project Online (несмотря на то, что некоторые функции, доступные только в Project Online).
  
После завершения переноса каждого должен проверить, чтобы убедиться в том, что он успешности переноса данных.
  
> [!NOTE]
> Если вы собираетесь только переход на Project Server 2013, если доступно не более локального решения, важно Обратите внимание на то, что она имеет только несколько несколько лет поддержки слева. Project Server 2013 с пакетом обновления 2 — Поддержка Дата окончания 10/13/2023. Дополнительные сведения о конечных дат поддержки можно [Политики жизненный цикл продукта Microsoft](https://go.microsoft.com/fwlink/p/?linkid=842066). 
  
### <a name="how-do-i-migrate-to-project-server-2019"></a>Как перенести Project Server 2019?

Архитектурные различия между Project Server 2010 и Project Server 2019 не позволяет способ прямой миграции. Это означает, что необходимо для переноса данных Project Server 2010 до следующей версии последовательных сервера Project Server, пока обновление до Project Server 2019.
  
Выполните следующие действия для обновления до Project Server 2019 будет необходимо:
  
1. Шаг 1: Перенос данных из Project Server 2010 до Project Server 2013.
    
2. Шаг 2: Перенос данных из Project 2013 обслуживать Project Server 2016.
    
3. Шаг 3: Перенос данных из Project Server 2016 Project Server 2019.
    
После завершения переноса каждого должен проверить, чтобы убедиться в том, что он успешности переноса данных.
