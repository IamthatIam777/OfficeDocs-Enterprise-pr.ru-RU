# <a name="accessible-diagram---design-sample-internet-sites-in-microsoft-azure-for-sharepoint-2013"></a>Доступная схема — пример проекта: веб-сайты в Microsoft Azure для SharePoint 2013

**Сводка:** Эта статья является доступным текстовая версия схемы с именем пример разработки: веб-сайты в Microsoft Azure для SharePoint 2013.
  
Используйте этот пример проекта в качестве отправной точки для использования в Интернете сайта в Azure с использованием SharePoint 2013.
  
На этом плакате показан пример того, как создать следующие аспекты SharePoint 2013:
  
- Пользователи
    
- Зоны и проверка подлинности
    
- Ферма серверов
    
- Сайт администрирования
    
- Службы
    
- Пулы приложений и веб-приложения
    
- Семейства веб-сайтов
    
- Сайты
    
- Базы данных контента
    
- Зоны и URL-адреса
    
## <a name="users-zones-and-authentication"></a>Пользователи, зоны и проверка подлинности

Этот проект включает учетные записи пользователей четырех типов. Каждый тип учетной записи связан с сайтом для доступа и зоной, использующей определенный тип проверки подлинности.  
  
- Анонимные пользователи — анонимные пользователи имеют доступ через сайта, например http://www.contoso.com. — Это зона, они используют «зона Интернета аудио- и анонимных», которая использует анонимный доступ.
    
- Проверка подлинности клиентов — имеют доступ заказчики проверкой подлинности через сайта, например https://secure.contoso.com. — Это зона, они используют «зона экстрасети / SAML», которая использует Azure Active Directory с проверкой подлинности SAML.
    
- Веб-сайтов авторов и разработчиков — веб-страниц и разработчики имеют доступ через сайты, такие как http://authoring.contoso.com:8000 или http://www.contoso.com:8000. — Это зона, они используют «зоны по умолчанию аудио- и встроенная Windows», с использованием доменных служб Active Directory (AD DS).
    
- Учетная запись обхода поиска — Учетная запись обхода поиска имеет доступ через сайты, такие как http://authoring.contoso.com:8000 или http://www.contoso.com:8000. — Это зона, она использует «зоны по умолчанию аудио- и встроенная Windows», которая использует Доменные службы Active Directory с проверкой подлинности Windows NTLM.
    
## <a name="server-farm"></a>Ферма серверов

Пользователи получают доступ к ферме серверов через службу Azure. Ферма серверов содержит один или несколько веб-серверов.
  
## <a name="administration-site"></a>Сайт администрирования

Сайт администрирования включает несколько серверов приложений, обменивающихся данными с пулом приложений (пулом приложений 1 в примере), который использует веб-приложение "Сайт центра администрирования". Сайт центра администрирования предоставляет доступ к семействам веб-сайтов в пределах организации.
  
Сайт администрирования также содержит серверы базы данных SQL, которые представляют из себя серверы базы данных с установленным сервером SQL Server, настроенным для поддержки кластеризации SQL, зеркального отображения или компонента AlwaysOn (последний используется только в SQL Server 2012).
  
## <a name="services"></a>Службы

В этом примере проекта показан веб-сайт служб IIS, веб-службы SharePoint. Веб-службы SharePoint включают пул приложений (пул приложений 2) с тремя службами (службами профилей пользователей, поиска и управляемых метаданных).
  
Примечания относительно служб для веб-сайтов
  
> Управляемые метаданные — Убедитесь, что для выбора **этого приложения-службы — это место хранения по умолчанию для наборов терминов, определяемых столбца**.
    
> Управление приложениями. Мы не рекомендуем использовать приложения на общедоступном веб-сайте в Azure.
    
## <a name="application-pools-and-web-applications"></a>Пулы приложений и веб-приложения

Группа по умолчанию в службе Azure включает пул приложений 3, содержащий веб-приложение с именем Contoso Sites. Это семейство веб-сайтов с именем на основе пути расположено по адресу http://internal:8000.
  
## <a name="site-collections-and-sites"></a>Семейства веб-сайтов и сайты

Семейства веб-сайтов в пуле приложений включают следующие компоненты:
  
- семейство веб-сайтов 1 с указанием узла в имени для обхода контента (пример расположения — http://authoring.contoso.com:8000);
    
- семейство веб-сайтов 2 с указанием узла в имени для запросов (примеры расположений — http://www.contoso.com, https://secure.contoso.com, http://www.contoso.com:8000);
    
- семейство веб-сайтов 3 с указанием узла в имени для запросов (примеры расположений — http://assets.contoso.com, https://secureassets.contoso.com, http://assets.contoso.com:8000).
    
## <a name="content-databases"></a>Базы данных контента

В этом примере показаны две базы данных контента. Одна предназначена для семейства веб-сайтов 1, используемого для обхода контента (http://authoring.contoso.com:8000). Вторая предназначена для двух семейств веб-сайтов 2 и 3, используемых для запросов (http://www.contoso.com, https://secure.contoso.com, http://www.contoso.com:8000, or http://assets.contoso.com, https://secureassets.contoso.com, http://assets.contoso.com:8000).
  
## <a name="zones-and-urls"></a>Зоны и URL-адреса

В этом примере приведены три зоны со связанными URL-адресами с балансировкой нагрузки, которые используются различными учетными записями.  
  
Первый список зон и URL-адресов связан с семейством веб-сайтов 1 и содержит следующую информацию:
  
- Пользователей — веб-страниц
    
- Зона — по умолчанию
    
- Подсистемы балансировки нагрузки URL-адрес - http://authoring.contoso.com:8000
    
Второй список зон и URL-адресов включает пользователей трех типов в трех различных зонах. Этот список связан с семейством веб-сайтов 2 и содержит следующую информацию.
  
Первая зона:
  
- Пользователей — веб-страниц
    
- Зона — по умолчанию
    
- Подсистемы балансировки нагрузки URL-адрес - http://www.contoso.com:8000
    
Вторая зона:
  
- Пользователи - анонимных пользователей
    
- Зона - Интернета
    
- Подсистемы балансировки нагрузки URL-адрес - http://www.contoso.com
    
Третья зона:
  
- Пользователи - прошедшего проверку подлинности клиентов
    
- Зона - экстрасети
    
- Подсистемы балансировки нагрузки URL-адрес - https://secure.contoso.com
    
Третий список зон и URL-адресов включает пользователей трех типов в трех различных зонах. Этот список связан с семейством веб-сайтов 3 и содержит следующую информацию.
  
Первая зона:
  
- Пользователей — веб-страниц
    
- Зона - Интернета
    
- Подсистемы балансировки нагрузки URL-адрес - http://assets.contoso.com:8000
    
Вторая зона:
  
- Пользователи - анонимных пользователей
    
- Зона - Интернета
    
- Подсистемы балансировки нагрузки URL-адрес - http://assets.contoso.com
    
Третья зона:
  
- Пользователи - прошедшего проверку подлинности клиентов
    
- Зона - экстрасети
    
- Подсистемы балансировки нагрузки URL-адрес - http://secureassets.contoso.com
