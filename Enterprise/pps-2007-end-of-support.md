## <a name="what-does-end-of-support-mean"></a>Окончание поддержки среднее какие работает?

Продукты Майкрософт, такие как PerformancePoint Server 2007 SP3, ProClarity программного обеспечения и SharePoint Server 2007 SP3 имелся жизненный цикл поддержки, во время которого корпорация Майкрософт предоставляет новые функции, исправления и обновления безопасности. Жизненный цикл продукта обычно длится десять лет с момента первоначального выпуска продукта и end, жизненный цикл называется окончание поддержки продукта. Как ProClarity, PerformancePoint Server и SharePoint Server 2007 достигшие их окончание поддержки, корпорация Майкрософт больше не предоставляет:
  
- Технической поддержки для решения проблем, которые могут возникнуть
    
- Исправления ошибок, проблем, обнаруженных и, которое может повлиять на надежность и удобство использования серверов
    
- Исправления безопасности для уязвимостей, обнаруженных и, которое может сделать серверов и приложений уязвимы к безопасности
    
- Обновления часового пояса
    
Установку ProClarity, SharePoint Server 2007 SP3 и PerformancePoint Server 2007 SP3 продолжит работу, несмотря на то, что более поддерживаются. Тем не менее настоятельно рекомендуется выполнить миграцию из этих приложений как можно скорее.

Как вы знаете, PerformancePoint Server 2007 был купить отдельно от SharePoint Server 2007. Приступая к работе с SharePoint Server 2010, службы PerformancePoint Services — это приложение службы в SharePoint Server. Это означает, что нет необходимости приобретение лицензий на отдельном сервере или оборудования для использования службы PerformancePoint Services.
  
Чтобы переместить PerformancePoint Server 2007 служб PerformancePoint Services, переместить в более позднюю версию SharePoint Server и настройка служб PerformancePoint Services. Версии SharePoint Server, перемещении определяет, будет ли импорт существующего контента панели мониторинга PerformancePoint Server 2007 для служб PerformancePoint Services.
  
- При обновлении до версии SharePoint Server 2010, можно импортировать контента панели мониторинга PerformancePoint с PerformancePoint Server 2007 для PerformancePoint Services в SharePoint Server 2010. Чтобы узнать больше о том, как это работает, обратитесь к разделу [Мастер импорта: контент PerformancePoint Server 2007 до SharePoint Server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- Если вы перемещаете для SharePoint Server 2013 или SharePoint Server 2016, вероятнее всего необходимо создать новое содержимое панели мониторинга (источники данных, отчеты, системы показателей и страниц панели мониторинга).
    
Чтобы начать работу с планом обновления служб PerformancePoint Services, можно найти в следующих ресурсах:
  
1. [Окончание поддержки стратегия SharePoint Server 2007](sharepoint-2007-end-of-support.md)
    
2. Если вы знаете, какие версии выполняется перенос в SharePoint, в статье соответствующий для служб PerformancePoint Services:
    
  - [Планирование служб PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [Обзор служб PerformancePoint Services в SharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Обзор служб PerformancePoint Services в SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
При обновлении до служб PerformancePoint Services понравится несколько новых функций и усовершенствований. PerformancePoint Services предлагает улучшенные систем показателей, новые визуализации, такие как дерево декомпозиции и отчета о ключевом показателе Эффективности и дополнительные типы диаграмм, улучшить возможности фильтрации операций со временем и улучшенная специальных возможностей. Для получения дополнительных сведений см [новые возможности служб PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841343).
