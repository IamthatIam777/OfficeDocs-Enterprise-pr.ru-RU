# <a name="project-server-2007-end-of-support-roadmap"></a>План действий после прекращения поддержки Project Server 2007;

Поддержка подходит к концу для серверов выпуска 2007 системы Office и приложений в 2017 и определите планов для миграции. При использовании в настоящее время Project Server 2007, обратите внимание на то, что он и этих связанных с ними продуктов будут иметь следующие даты окончания поддержки:
  
|**Продукт**|**Поддержка Дата окончания**|
|:-----|:-----|
|Project Server 2007  <br/> |10 октября 2017 г.  <br/> |
|Project Portfolio Server 2007  <br/> |10 октября 2017 г.  <br/> |
|Project Стандартный 2007  <br/> |10 октября 2017 г.  <br/> |
|Project профессиональный 2007  <br/> |10 октября 2017 г.  <br/> |
   
Дополнительные сведения о достигает пенсионного обеспечения серверов выпуска 2007 системы Office содержатся в разделе [обновление от серверов выпуска 2007 системы Office и продуктами клиента](upgrade-from-office-2007-servers-and-products.md).
  
## <a name="what-does-end-of-support-mean"></a>Окончание поддержки среднее какие работает?

Project Server, как практически во всех продуктов Майкрософт, имеет жизненный цикл поддержки, во время которого предоставляются новые функции, исправления ошибок, исправления безопасности и т. д. Обычно этот жизненный цикл продолжается десять лет с момента первоначального выпуска продукта и конечных этот жизненный цикл называется окончание поддержки продукта. Так как Project Server 2007 связаться с вами его окончание поддержки на 10 октября 2017, больше не будет предоставлять корпорации Майкрософт:
  
- Техническая поддержка проблем, которые могут возникнуть.
    
- Наличие проблем, обнаруженных и, которое может повлиять на надежность и удобство использования сервера исправления ошибок.
    
- Исправления безопасности для уязвимостей, обнаруженных и то может привести к серверу уязвимы к безопасности.
    
- Обновления для часового пояса.
    
Установки Project Server 2007 будут выполняться после указанной даты. Тем не менее из-за изменения, перечисленные выше, мы настоятельно рекомендуем выполнить миграцию из Project Server 2007 как можно скорее.

- Перенос в Project Online
    
- Миграция на новую версию локального сервера Project Server (желательно 2016 Project Server).
    
|**Почему предпочитают перенос в Project Online**|**Почему предпочитают перенос в Project Server 2016**|
|:-----|:-----|
| У меня есть пользователей мобильных устройств.  <br/>  Расходы на перенос являются большой проблемой (оборудование, программное обеспечение, часов и меры по реализации, и т.д.).  <br/>  После миграции расходы на обслуживание моей среды являются большой проблемой (например, для автоматического обновления, гарантируется время работы, и т.д.).  <br/> | Бизнес-правил ограничить мне операционной бизнес в облаке.  <br/>  Требуется элемент управления обновлений моей среды.  <br/> |
   
> [!NOTE]
> Дополнительные сведения о параметрах перемещения с серверов Office 2007 можно [ресурсы, которые помогут вам обновление с Office 2007 серверов и клиентов](upgrade-from-office-2007-servers-and-products.md). Обратите внимание на то, что Project Server не поддерживает гибридной конфигурации, начиная с Project Server и Project Online не могут совместно использовать тот же самый пул ресурсов. 
  
## <a name="important-considerations-you-need-to-make-when-planning-to-migrate-from-project-server-2007"></a>Важные сведения, которые необходимо принять при планировании для переноса из Project Server 2007

Вам необходимо при планировании для переноса из Project Server 2007, учтите следующее:
  
- **Получите помощь от партнера корпорации Майкрософт** - обновление из Project Server 2007 может оказаться сложной задачей и требует большого количества планирования и подготовки. Его может быть особенно сложной, если не было необходимо установить и настроить Project Server 2007 изначально. К счастью существует партнеров корпорации Майкрософт, можно включить для пользователей, которые этого на жизнь ли планируется перенос Project Server 2016 или Project Online. Можно выполнить поиск партнера корпорации Майкрософт при миграции на [Центр партнеров Майкрософт](https://go.microsoft.com/fwlink/p/?linkid=841249). Поиск по ключевым словам на термин *уровня Gold проекта и управления портфелем* можно извлечь список всех Microsoft партнеров с опытом в проекте. 
    
- **Планирование настройки** - помните о том, что многие из настроек, у вас есть работа в вашей среде Project Server 2007 может не работать при миграции для Project Server 2016 или Project Online. Существует большая различия в архитектуре Project Server версии, а также обязательные операционных систем, серверов баз данных и клиентского веб-браузеров, поддерживаемых для работы с новой версией. Составить план на месте на инструкции по тестированию или перестроение настройки при необходимости в новой среде. Планирование обновления также будут хорошая возможность для проверки, если определенные настройки действительно требуется при переходе. [Создание плана текущих настроек во время обновления до SharePoint 2013](https://go.microsoft.com/fwlink/p/?linkid=842061) имеет некоторые хорошие Общие сведения о оценки и планирования текущих настроек при обновлении. 
    
- **Время и Терпение** - обновление планирование, выполнение и тестирование займет много времени и усилий, особенно при обновлении до Project Server 2016. К примеру при переносе из Project Server 2007 для Project Server 2016, будет сначала необходимо переноса из Project Server 2007 до Project Server 2010, а затем проверьте данные и выполните одинаковые значения, при переходе на всех последующих версий. Можно проверить с помощью партнера корпорации Майкрософт для сравнения расходы с их оценки как время, необходимое для их действий и какова стоимость.

Можно вручную создать конфигурацию веб-клиента Project в Project Online (например, воссоздайте все необходимые дополнительные поля или корпоративные календари). Партнеры корпорации Майкрософт также могут помочь вам с это.

## <a name="migrate-to-a-newer-on-premises-version-of-project-server"></a>Миграция на новую версию локального сервера Project Server

Хотя мы настоятельно считаем, что можно достичь оптимального значение и пользователя, перенос в Project Online, также понять, что в некоторых организациях требуется хранить данные проекта в локальной среде. Если нужно сохранить проект данных локального может перенести среды Project Server 2007 для Project Server 2010, Project Server 2013 или Project Server 2016.
  
Мы рекомендуем перенос в Project Server 2016, если не удается выполнить миграцию в Project Online. Project Server 2016 включает все функции и усовершенствования, включенные в предыдущие версии Project Server и наиболее близко соответствует удобство работы с Project Online (несмотря на то, что некоторые функции, доступные только в Project Online).
  
После завершения переноса каждого должен проверить, чтобы убедиться в том, что он успешности переноса данных.
  
> [!NOTE]
> Если вы собираетесь только перенос в Project Server 2010, если доступно не более локального решения, важно Обратите внимание на то, что она имеет только несколько несколько лет поддержки слева. Project Server 2010 с пакетом обновления 2 — Поддержка Дата окончания 10/13/2020 г. Дополнительные сведения о конечных дат поддержки можно [Политики жизненный цикл продукта Microsoft](https://go.microsoft.com/fwlink/p/?linkid=842066). 
  
### <a name="how-do-i-migrate-to-project-server-2016"></a>Как перенести Project Server 2016?

Архитектурные различия между Project Server 2007 и Project Server 2016 не позволяет способ прямой миграции. Это означает, что необходимо для переноса данных Project Server 2007 до следующей версии последовательных сервера Project Server, пока обновление до Project Server 2016.
  
Выполните следующие действия для обновления до Project Server 2016 будет необходимо:
  
1. Шаг 1: Перенос из Project Server 2007 до Project Server 2010.
    
2. Шаг 2: Перенос из служебной Project 2010 до Project Server 2013.
    
3. Шаг 3: Перенос из Project Server 2013 в Project Server 2016.
    
После завершения переноса каждого должен проверить, чтобы убедиться в том, что он успешности переноса данных.

#### <a name="how-do-i-upgrade"></a>Как обновление?

Во время сведения об обновлении можно найти в набор содержимого [обновления до Project Server 2010](https://go.microsoft.com/fwlink/p/?linkid=841812) , важно понимать, что две различные методы, которые можно использовать для обновления: 
  
- **Обновление с подключением базы данных:** Этот метод только обновляет контент для вашей среды, а не параметры конфигурации. Он является обязательным при обновлении Office Project Server 2007 развернут на оборудование, которое поддерживает только 32-разрядная версия серверная операционная система. Существует два типа методов обновления с переподключением баз данных: 
    
  - **Полное обновление с подключением базы данных** — выполняется перенос данных проекта из баз данных Office Project Server 2007, а также данных сайта Microsoft Project Web App (PWA), хранящиеся в базе данных контента SharePoint. 
    
  - **Database-attach Core upgrade** - выполняется перенос только данных проекта из баз данных Project Server. 
    
- **Обновление на месте**: данные конфигурации для фермы и все содержимое в ферме выполняется обновление на имеющегося оборудования в определенном порядке. Перед запуском процесса обновления на месте, программа установки использует всей фермы в автономном режиме и веб-сайтов и сайты Microsoft Project Web App недоступны до завершения обновления и установки перезапуск сервера. После начала обновления на месте не удается приостановить обновления или откат к предыдущей версии. Настоятельно рекомендуется использовать зеркальное копирование образа производственной среды и выполнить обновление на месте для этой среды, а не производственной среды.
