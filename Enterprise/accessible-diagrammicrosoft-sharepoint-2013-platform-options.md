# <a name="accessible-diagram---microsoft-sharepoint-2013-platform-options"></a>Доступная схема — варианты платформы Microsoft SharePoint 2013

**Сводка:** Эта статья представляет собой текстовую версию схемы, именуемую вариантами платформы Microsoft SharePoint 2013.
  
Какие лица, принимающие решения в Организации (вариантах развертывания) и архитекторы, должны знать о разрешениях для Office 365, Microsoft Azure и локальных развертываний. 
  
Этот плакат состоит из двух следующих разделов. 
  
- Сравнение четырех различных развертываний для SharePoint 2013: SharePoint в Office 365, гибридной среды Office 365 с локальным развертыванием SharePoint 2013, Azure и локальным развертыванием SharePoint 2013. 
    
- Описание трех типичных рабочих нагрузок для перемещения в Azure. 
    
## <a name="comparison-of-four-different-deployments-for-the-sharepoint-2013-platform"></a>Сравнение четырех различных вариантов развертывания платформы SharePoint 2013

В сравнении по каждому варианту развертывания представлена следующая информация: 
  
- обзор различных компонентов развертывания; 
    
- рекомендации по оптимальному использованию каждого типа развертывания;  
    
- Требования лицензирования 
    
- архитектурные задачи, необходимые для реализации решения;  
    
- обязанности ИТ-специалистов в отношении реализации решения.  
    
### <a name="overview"></a>Общие сведения

#### <a name="sharepoint-2013-in-office-365"></a>SharePoint 2013 в Office 365

Повысить эффективность и оптимизировать стоимость с помощью планов Office 365 для нескольких клиентов. 
  
На сопроводительной схеме показана среда SharePoint Online с клиентом Azure Active Directory, которая синхронизирует имена и пароли учетных записей между локальной средой Active Directory и клиентом Azure Active Directory. 
  
Описание функций 
  
- Программное обеспечение как услуга (SaaS). 
    
- Широкий спектр функций постоянно обновляется.  
    
- Включает клиент Azure Active Directory (может использоваться с другими приложениями). 
    
- Интеграция каталогов включает синхронизацию имен учетных записей и паролей между локальной средой Active Directory и клиентом Azure Active Directory. 
    
- Если необходима функция единого входа, можно реализовать службы федерации Active Directory (AD FS).   
    
- Связь с клиентом через Интернет шифруется и сопровождается проверкой подлинности (через порт 443).  
    
- Перенос данных ограничен данными, которые можно отправить через Интернет.  
    
- Настройки: приложения для Office и SharePoint, SharePoint Designer 2013.  
    
#### <a name="hybrid-with-office-365"></a>Гибридная среда с Office 365

Объедините преимущества Office 365 с локальным развертыванием SharePoint 2013. 
  
На сопроводительной схеме показано Office 365 с SharePoint Online с помощью Business Connectivity Services (BCS) для подключения к локальной ферме SharePoint Server 2013. 
  
Выберите функцию для интеграции из перечисленных ниже.  
  
Поиск SharePoint 
  
- Пользователи могут просматривать результаты поиска в обеих средах.   
    
- Пользователи экстрасети могут выполнять удаленный вход в локальную учетную запись Active Directory и пользоваться всеми доступными функциями гибридной среды.  
    
BCS
  
Из SharePoint Online: пользователи могут выполнять операции чтения и записи. Служба BCS подключается к локальной ферме SharePoint Server 2013. Служба BCS, настроенная на локальной ферме, выступает в роли посредника соединения с конечными точками локальной службы OData.   
  
Duet Enterprise Online 
  
Из SharePoint Online пользователи могут выполнять операции чтения и записи по отношению к локальной системе SAP.  
  
#### <a name="azure"></a>Azure

Пользуйтесь облачными службами, сохраняя при этом полный контроль над платформой и компонентами.   
  
На сопроводительной схеме показана служба Azure, которая содержит две облачные службы, ферму SharePoint 2013 и Windows Server Active Directory с DNS-подключением к пользователям через Интернет или подключением к локальной службе Active Directory через VPN-туннель. 
  
Поддерживаются перечисленные ниже возможности.  
  
-  Azure — это платформа, предоставляющая инфраструктуру и службы приложений, необходимые для размещения фермы SharePoint 2013.
    
- Службы инфраструктуры. 
    
- Лучшая собственная облачная платформа для SQL Server и SharePoint.  
    
- Вычислительные ресурсы доступны практически сразу без каких-либо обязательств.  
    
- Сосредоточьтесь на приложениях, а не центрах обработки данных и инфраструктуре.  
    
- Недорогие среды разработки и тестирования.  
    
- Решения SharePoint могут быть доступны из Интернета или только из корпоративной среды через VPN-туннель типа "сеть-сеть".  
    
- Спектр настроек неограничен.  
    
#### <a name="on-premises"></a>Локальная среда

Все под вашим контролем.   
  
На сопроводительной схеме показана локальная среда с веб-серверами, серверами приложений и службой Active Directory, которая обменивается контентом со всеми базами данных и серверами приложений, выделенными для поиска.  
  
Поддерживаются перечисленные ниже возможности.  
  
- Планирование загрузки и определение конфигурации оборудования. 
    
- Приобретение и настройка серверов.  
    
- Развертывание. 
    
- Горизонтальное масштабирование, применение исправлений и эксплуатация.   
    
- Резервное копирование данных. 
    
- Поддержка среды аварийного восстановления.   
    
- Спектр настроек неограничен.

- Web Analytics.

- Устаревшие решения со сторонними компонентами, которые зависят от аппаратного и программного обеспечения, которые не поддерживаются в службах инфраструктуры Azure. 
    
- Ограничения конфиденциальности, которые предотвращают синхронизацию учетных записей Active Directory с Azure Active Directory (требование для Office 365).

## <a name="three-typical-workloads-to-move-to-azure"></a>Три типичных рабочих нагрузки для перемещения в Azure

### <a name="office-365-plus-directory-components-in-azure"></a>Office 365 Plus компоненты каталога в Azure

Развертывание компонентов интеграции каталогов Office 365 в Azure выполняется быстрее, так как он позволяет развертывать виртуальные машины по запросу. 
  
#### <a name="directory-synchronization-server-only"></a>Только сервер синхронизации службы каталогов

Вместо того чтобы развертывать 64 – разрядный сервер синхронизации каталогов в локальной среде, необходимо подготовить виртуальную машину в Azure. 
  
На сопроводительной схеме показана среда SharePoint Online с клиентом Azure Active Directory, которая синхронизирует имена и пароли учетных записей между локальной средой Active Directory и клиентом Azure Active Directory. 
  
#### <a name="directory-synchronization-plus-ad-fs"></a>Синхронизация службы каталогов и службы AD FS

Этот вариант позволяет поддерживать федеративные удостоверения Office 365, не добавляя оборудование в локальную инфраструктуру. Кроме того, при использовании этого варианта обеспечивается устойчивость системы в случае недоступности локальной среды Active Directory. 
  
- Компоненты интеграции каталогов хранятся в Azure. 
    
- СЛУЖБЫ AD FS публикуются в Интернете через прокси-серверы AD FS в Azure. 
    
- Трафик проверки подлинности клиента для пользователей, подключающихся из любого расположения, обрабатывается серверами AD FS и прокси-серверами, развернутыми в Azure. 
    
### <a name="public-facing-internet-site-and-azure-ad-for-customer-authentication"></a>ОбщеДоступный Интернет-сайт и Azure AD для проверки подлинности клиентов

Воспользуйтесь возможностью легко масштабироваться по запросу, разместив сайт Интернета в Azure. Используйте Azure AD для хранения учетных записей клиентов. 
  
#### <a name="azure-advantages-for-internet-sites"></a>Преимущества Azure для Интернет-сайтов

- Вы платите только за необходимые ресурсы, изменяя количество виртуальных машин в зависимости от того, насколько интенсивно используется ферма.  
    
- Вы можете добавить подробную отчетность и веб-аналитику. 
    
- Вы можете сосредоточиться на разработке хорошего сайта, а не создании инфраструктуры. 
    
#### <a name="azure-ad"></a>Azure AD

Azure AD предоставляет возможности управления удостоверениями и управления доступом для облачных служб. Возможности включают облачное хранилище для данных каталога и основной набор служб удостоверений, в том числе процессы входа пользователей, службы проверки подлинности и службы федерации Active Directory. Службы удостоверений, включенные в Azure AD, легко интегрируются с локальными развертываниями Active Directory и полностью поддерживают сторонние поставщики удостоверений. 
  
На сопроводительной схеме показана конфигурация зон и проверки подлинности, важная для общедоступных веб-сайтов. На схеме показан клиент Azure Active Directory, который содержит ферму SharePoint в Azure с двумя зонами: 
  
- зоной Интернета, взаимодействующей с анонимными и прошедшими проверку подлинности пользователями и клиентами, которые не входят в сеть;  
    
- зоной по умолчанию зоны, которая содержит NTLM для обхода контента и проверки подлинности Windows, а также взаимодействует с локальным развертыванием Active Directory через VPN-туннель.   
    
### <a name="on-premises-farm-plus-disaster-recovery-in-azure"></a>Локальная ферма и аварийное восстановление в Azure

Выберите вариант аварийного восстановления, который соответствует требованиям вашей компании. Azure предоставляет варианты уровня ввода для компаний, которые приступит к работе с аварийным восстановлением и дополнительными возможностями для предприятий с высокими требованиями к устойчивости. 
  
#### <a name="cold-standby"></a>Холодное резервирование

- Ферма создается в полном масштабе, но виртуальные машины останавливаются. (Вы оплачиваете только при выполнении!) 
    
- Обслуживание среды включает запуск виртуальных машин по мере необходимости, а также исправление, обновление и проверку среды. 
    
- Полный запуск среды в случае сбоя. 
    
#### <a name="warm-standby"></a>Резервный сервер

- Включает небольшую ферму, которая уже подготовлена к работе и запущена.   
    
- При отработке отказа эта ферма может мгновенно обеспечить обслуживание пользователей.   
    
- Быстрое масштабирование фермы для обслуживания всех пользователей.  
    
#### <a name="hot-standby"></a>Сервер горячей замены

Полноразмерная ферма подготавливается к работе и запускается в ждущем режиме. 
  
На сопроводительной схеме показана локальная ферма, которая взаимодействует с средой аварийного восстановления SharePoint в Azure. Локальные базы данных используют доставку журналов SQL Server через VPN-туннель для доступа к среде аварийного восстановления SharePoint, включающей два сервера баз данных SQL, которые содержат базы данных SharePoint, два веб-сервера переднего плана и два сервера приложений SharePoint.
