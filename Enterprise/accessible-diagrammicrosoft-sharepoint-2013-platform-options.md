**Сводка:** Эта статья является доступным текстовая версия схемы с именем параметры платформе Microsoft SharePoint 2013.
  
Что business лиц, принимающих решения (СУБД) и создают нужно знать об Office 365, Microsoft Azure и локальные развертывания. 
  
Этот плакат состоит из двух следующих разделов. 
  
- Сравнение четыре разных развертываниях для SharePoint 2013: SharePoint в Office 365 и гибридных Office 365 с локальным развертыванием SharePoint 2013, Azure и локальное развертывание SharePoint 2013. 
    
- Описание трех типичных рабочих нагрузок для перемещения Azure.

В сравнении по каждому варианту развертывания представлена следующая информация: 
  
- обзор различных компонентов развертывания;

### <a name="overview"></a>Общие сведения

#### <a name="sharepoint-2013-in-office-365"></a>SharePoint 2013 в Office 365

Повышения эффективности и оптимизация для затрат с помощью планов обслуживания Office 365. 
  
Сопутствующая схеме показана SharePoint Online с помощью клиента Azure Active Directory, которая синхронизирует имена учетных записей и пароли между локальной среды Active Directory и клиента Azure Active Directory. 
  
Описание функций 
  
- Программное обеспечение как услуга (SaaS). 
    
- Широкий спектр функций постоянно обновляется.  
    
- Включает в себя Azure Active Directory клиента (используется вместе с другими приложениями). 
    
- Интеграция каталогов включает в себя синхронизация имена учетных записей и пароли между локальной среды Active Directory и клиента Azure Active Directory. 
    
- Если необходима функция единого входа, можно реализовать службы федерации Active Directory (AD FS).   
    
- Связь с клиентом через Интернет шифруется и сопровождается проверкой подлинности (через порт 443).  
    
- Перенос данных ограничен данными, которые можно отправить через Интернет.  
    
- Настройки: приложения для Office и SharePoint, SharePoint Designer 2013.  
    
#### <a name="hybrid-with-office-365"></a>Гибридное развертывание с Office 365

Преимущества Office 365 объединить с локальным развертыванием SharePoint 2013. 
  
На соответствующие схеме показана Office 365 с помощью SharePoint Online с помощью Business Connectivity Services (BCS), чтобы подключиться к локальной ферме SharePoint Server 2013. 
  
Выберите функцию для интеграции из перечисленных ниже.  
  
Поиск SharePoint 
  
- Пользователи могут просматривать результаты поиска в обеих средах.   
    
- Пользователи экстрасети могут выполнять удаленный вход в локальную учетную запись Active Directory и пользоваться всеми доступными функциями гибридной среды.  
    
BCS
  
Из SharePoint Online: пользователи могут выполнять операции чтения и записи. Служба BCS подключается к локальной ферме SharePoint Server 2013. Служба BCS, настроенная на локальной ферме, выступает в роли посредника соединения с конечными точками локальной службы OData.   
  
Duet Enterprise Online 
  
Из SharePoint Online пользователи могут выполнять операции чтения и записи по отношению к локальной системе SAP.  
  
#### <a name="azure"></a>Azure

Пользуйтесь облачными службами, сохраняя при этом полный контроль над платформой и компонентами.   
  
Сопутствующая схеме показана Azure, который содержит два облачные службы, фермы SharePoint 2013 и Windows Server Active Directory с DNS соединение для пользователей через Интернет или подключиться к локальной службе Active Directory через VPN-туннеля. 
  
Поддерживаются перечисленные ниже возможности.  
  
-  Azure — это платформа, которая предоставляет услуги инфраструктуры и приложения, необходимые для размещения в ферме SharePoint 2013.
    
- Службы инфраструктуры. 
    
- Лучшая собственная облачная платформа для SQL Server и SharePoint.  
    
- Вычислительные ресурсы доступны практически сразу без каких-либо обязательств.  
    
- Сосредоточьтесь на приложениях, а не центрах обработки данных и инфраструктуре.  
    
- Недорогие среды разработки и тестирования.  
    
- Решения SharePoint могут быть доступны из Интернета или только из корпоративной среды через VPN-туннель типа "сеть-сеть".  
    
- Спектр настроек неограничен.  
    
#### <a name="on-premises"></a>Локальная среда

Все под вашим контролем.   
  
На сопроводительной схеме показана локальная среда с веб-серверами, серверами приложений и службой Active Directory, которая обменивается контентом со всеми базами данных и серверами приложений, выделенными для поиска.  
  
Поддерживаются перечисленные ниже возможности.

- Приобретение и настройка серверов. 
    
- Развертывание. 
    
- Горизонтальное масштабирование, применение исправлений и эксплуатация. 
    
- Резервное копирование данных. 
    
- Поддержка среды аварийного восстановления.   
    
- Спектр настроек неограничен.

## <a name="three-typical-workloads-to-move-to-azure"></a>Три стандартные рабочие нагрузки для перемещения в Azure

### <a name="office-365-plus-directory-components-in-azure"></a>Office 365, а также компоненты каталогов в Azure

Развертывание компонентов интеграции каталогов Office 365 в Azure работает быстрее, так как его можно развернуть виртуальных машин по запросу. 
  
#### <a name="directory-synchronization-server-only"></a>Только сервер синхронизации службы каталогов

Вместо развертывание сервера синхронизации каталогов 64-разрядная версия в локальной среде, вместо этого временного виртуальной машины в Azure. 
  
Сопутствующая схеме показана SharePoint Online с помощью клиента Azure Active Directory, которая синхронизирует имена учетных записей и пароли между локальной среды Active Directory и клиента Azure Active Directory. 
  
#### <a name="directory-synchronization-plus-ad-fs"></a>Синхронизация службы каталогов и службы AD FS

Этот параметр используется для поддержки Office 365 федеративных удостоверений (SSO) без добавления оборудования в локальной инфраструктуре. Он также предоставляет устойчивости недоступность локальной среды Active Directory. 
  
- Компоненты интеграции каталогов размещаются в Azure. 
    
- Службы федерации Active Directory будет опубликована в Интернет через прокси-серверы AD FS в Azure. 
    
- Трафика проверки подлинности клиента, для пользователей, подключающихся из любого места обрабатывается серверов AD FS и прокси-серверы, развернутые в Azure. 
    
### <a name="public-facing-internet-site-and-azure-ad-for-customer-authentication"></a>Общедоступный веб-сайтом и Azure AD для проверки подлинности клиента

Воспользуйтесь преимуществами возможность легко масштабирования к запросу, поместив веб-сайт в Azure. Используйте Azure AD для хранения учетных записей клиентов. 
  
#### <a name="azure-advantages-for-internet-sites"></a>Azure преимущества для веб-сайтов

- Вы платите только за необходимые ресурсы, изменяя количество виртуальных машин в зависимости от того, насколько интенсивно используется ферма.  
    
- Вы можете добавить подробную отчетность и веб-аналитику. 
    
- Вы можете сосредоточиться на разработке хорошего сайта, а не создании инфраструктуры. 
    
#### <a name="azure-ad"></a>Azure AD

Azure AD предоставляет identity management и доступа к возможности управления для облачных служб. Возможности включают облачной хранилища данных каталога и базовый набор удостоверения службы, включая процессы входа пользователя, службы проверки подлинности и службы федерации Active Directory. Удостоверение службы, которые включены в Azure AD интегрируются с развертываниями Active Directory вашей локальной и полностью поддерживает удостоверения сторонних поставщиков. 
  
Сопутствующая схеме показана конфигурация зон и проверки подлинности, важные для интернет-сайтов. На схеме показана Azure Active Directory клиента, который содержит ферму SharePoint на Azure с двумя зонами: 
  
- зоной Интернета, взаимодействующей с анонимными и прошедшими проверку подлинности пользователями и клиентами, которые не входят в сеть;  
    
- зоной по умолчанию зоны, которая содержит NTLM для обхода контента и проверки подлинности Windows, а также взаимодействует с локальным развертыванием Active Directory через VPN-туннель.   
    
### <a name="on-premises-farm-plus-disaster-recovery-in-azure"></a>Локальная ферма, а также аварийное восстановление в Azure

Выберите вариант аварийного восстановления, которая соответствует бизнес-требований. Azure предоставляет базовое параметры для организаций, Приступая к работе с аварийного восстановления и дополнительные параметры для предприятий, требования к высокой устойчивости. 
  
#### <a name="cold-standby"></a>Холодное резервирование

- Полностью построения фермы, но остановлен виртуальных машин. (Вы оплаты только когда они работают!) 
    
- Обслуживание среды включает запуск виртуальных машин по мере необходимости, а также исправление, обновление и проверку среды. 
    
- Полный запуск среды в случае сбоя. 
    
#### <a name="warm-standby"></a>Резервный сервер

- Включает небольшую ферму, которая уже подготовлена к работе и запущена.   
    
- При отработке отказа эта ферма может мгновенно обеспечить обслуживание пользователей.   
    
- Быстрое масштабирование фермы для обслуживания всех пользователей.  
    
#### <a name="hot-standby"></a>Сервер горячей замены

Полноразмерная ферма подготавливается к работе и запускается в ждущем режиме. 
  
Сопутствующая схеме показана локальная ферма, взаимодействует со средой SharePoint аварийное восстановление в Azure. Баз данных в локальной использование доставки журналов SQL Server через VPN-туннеля для доступа к среды SharePoint аварийного восстановления, которая включает в себя два сервера баз данных SQL, которые содержат баз данных SharePoint, два сервера веб-интерфейс и два SharePoint Серверы приложений.
